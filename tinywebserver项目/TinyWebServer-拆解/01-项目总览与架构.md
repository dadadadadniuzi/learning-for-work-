# 项目总览与架构

返回：[[TinyWebServer-面试拆解笔记]]

相关：[[TinyWebServer-拆解/02-请求链路与从零实现]]、[[TinyWebServer-拆解/03-入口与WebServer总控]]

## 一句话讲清项目

TinyWebServer 是一个运行在 Linux 上的轻量级 C++ Web 服务器。它基于 `socket + epoll + 非阻塞 I/O + 线程池 + 状态机` 处理 HTTP 请求，并实现了：

- 静态资源访问
- 用户注册和登录
- MySQL 数据库连接池
- 空闲连接超时回收
- 同步和异步日志
- Reactor 和 Proactor 两种处理模型

## 用户能看到什么

- 浏览器访问首页和静态页面
- 请求图片和视频文件
- 注册账号
- 登录账号
- 登录成功后进入欢迎页

## 面试官真正关心什么

- 服务器如何监听端口并接收连接
- 如何高并发处理多个请求
- HTTP 报文如何解析
- 线程池如何分发任务
- 数据库连接如何复用
- 超时连接如何关闭
- 日志如何不拖慢主流程

## 架构图和源码的对应关系

- `main.cpp`
  程序入口
- `config.h / config.cpp`
  命令行参数解析
- `webserver.h / webserver.cpp`
  服务器总控，负责主线程、epoll、监听 socket、信号和定时器调度
- `http/http_conn.h / http_conn.cpp`
  单个 HTTP 连接的读、解析、处理和响应
- `threadpool/threadpool.h`
  工作线程池和任务队列
- `timer/lst_timer.h / lst_timer.cpp`
  超时定时器和空闲连接回收
- `CGImysql/sql_connection_pool.h / sql_connection_pool.cpp`
  MySQL 连接池
- `log/log.h / log.cpp / block_queue.h`
  同步和异步日志
- `lock/locker.h`
  互斥锁、条件变量、信号量封装
- `root/`
  网站静态资源目录

## 整体设计思路

这个项目的核心思想是：

- 主线程轻量化
  主线程只做事件监听、接收连接和任务分发
- 工作线程专注处理任务
  读写、解析和业务逻辑都交给线程池
- 资源复用
  数据库连接使用连接池复用
- 超时治理
  空闲连接通过定时器回收
- 可观测性
  日志支持同步和异步模式

## 你可以怎么开场介绍

> 这是一个以 Linux 高并发网络编程为核心的 C++ Web 服务器项目。主线程用 epoll 做事件监听和分发，工作线程通过线程池处理请求；单个连接由 http_conn 对象维护，HTTP 报文通过状态机解析；数据库访问使用 MySQL 连接池；空闲连接通过定时器回收；日志支持同步和异步两种模式。业务上实现了静态资源访问、用户注册和登录。
