# telnet

## 它是什么

`telnet` 是一个可以手工建立 TCP 连接并直接输入文本数据的命令行工具。

在网络编程学习里，它常用来：

- 测试自己写的 TCP 服务器是否能连通。
- 手工向服务器发送一段文本。
- 手工构造简单的 HTTP 请求，观察服务器返回内容。

## 常见写法

```bash
telnet 127.0.0.1 9527
telnet 127.0.0.1 8080
```

## 参数说明

- `目标IP`：要连接的服务器地址，例如 `127.0.0.1` 表示本机。
- `目标端口`：服务器监听的端口，例如 `9527` 或 `8080`。

## 在 TCP 测试中的作用

如果你写了一个 TCP 服务器，例如监听 `9527` 端口，可以执行：

```bash
telnet 127.0.0.1 9527
```

连接成功后，在终端输入文本，服务器就能收到这些字节。如果服务器实现了回显、大小写转换等逻辑，也可以直接看到返回结果。

## 在 HTTP 测试中的作用

如果你写了一个简单 Web 服务器监听 `8080` 端口，可以手工发送 HTTP 请求：

```bash
telnet 127.0.0.1 8080
```

连接成功后输入：

```text
GET /index.html HTTP/1.1
Host: 127.0.0.1

```

注意：

- 最后一行空行非常重要，表示请求头结束。
- 发送完成后，服务器通常会返回完整的 HTTP 响应报文。

这非常适合学习 [[linux网络编程/概念词条/HTTP请求报文|HTTP 请求报文]] 和 [[linux网络编程/概念词条/HTTP响应报文|HTTP 响应报文]] 的真实样子。

## 和 nc / curl 的区别

- `telnet`：适合手工一行一行输入文本，观察交互过程。
- [[linux网络编程/指令查询/命令卡/nc|nc]]：更通用，也常用于 TCP/UDP 测试。
- [[linux网络编程/指令查询/命令卡/curl|curl]]：更适合测试 HTTP，能自动帮你构造完整请求。

可以这样记：

- 想手工输协议文本：优先想到 `telnet`。
- 想快速测端口或发简单文本：想到 `nc`。
- 想更像真实 HTTP 客户端去测 Web 服务器：想到 `curl`。

## 易错点

- `telnet` 只能直接拿来测 TCP，不适合像 `nc -u` 那样测试 UDP。
- 连接失败不一定是代码错误，也可能是端口没监听、IP 写错、防火墙拦截。
- 测 HTTP 时，最后必须有空行，否则服务器可能一直在等请求头结束。
- 有些系统默认没有安装 `telnet`。
