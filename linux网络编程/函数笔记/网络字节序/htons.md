# htons

> [!info] 功能
> 将 16 位整数从主机字节序转换为[[linux网络编程/概念词条/网络字节序|网络字节序]]。

## 函数原型

- `uint16_t htons(uint16_t hostshort);`

## 依赖头文件

- `#include <arpa/inet.h>`

## 输入参数

- `hostshort`：主机字节序下的 16 位无符号整数。网络编程中最常用于[[linux网络编程/概念词条/端口|端口]]号，例如 `9527`、`8080`。传入前通常是人正常书写的端口整数。

## 输出参数

- 无直接输出参数。

## 返回值

- 返回转换后的 16 位网络字节序整数。

## 知识点补充

- `sin_port` 字段要求网络字节序，所以几乎总是写成 `htons(port)`。
- `s` 表示 short，端口号正好是 16 位。

## 常见用法

```c
servaddr.sin_port = htons(9527);
```

## 易错点

- 不要直接写 `servaddr.sin_port = 9527;`。
- 如果打印网络结构里的端口，要用 [[linux网络编程/函数笔记/网络字节序/ntohs|ntohs]] 转回主机字节序。
