# 多线程并发服务器

## 它是什么

- 多线程并发服务器是主线程负责接收连接，每个客户端连接交给一个工作线程处理的服务器模型。

## 怎么理解

- 线程比进程更轻量。
- 同一进程内的线程共享地址空间和文件描述符表。
- 工作线程可以直接使用传入的 `cfd` 和客户端通信。

## 典型流程

- 主线程创建、绑定、监听 socket。
- 主线程循环 `accept`。
- 每接收一个客户端，创建工作线程。
- 工作线程处理读写。
- 工作线程结束前关闭 `cfd`。
- 如果不 `join`，通常使用 `pthread_detach` 分离线程。
- 如果连接很多，可以进一步使用 [[linux网络编程/概念词条/线程池|线程池]]，提前创建固定工作线程，通过任务队列分发连接。

## 优点

- 创建成本比进程低。
- 线程之间共享数据更方便。

## 缺点

- 共享数据容易引入竞态，需要锁或其他同步机制。
- 传参时要避免多个线程共用同一个被覆盖的局部变量地址。
- 每连接一个线程在高并发下可能导致线程数量过多，此时应考虑 [[linux网络编程/概念词条/线程池|线程池]] 或 [[linux网络编程/概念词条/IO多路复用|IO多路复用]]。
