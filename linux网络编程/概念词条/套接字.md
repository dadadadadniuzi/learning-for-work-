# 套接字

## 它是什么

- 套接字是应用程序访问内核网络协议栈的接口。
- 在 Linux 中，调用 `socket` 成功后会返回一个文件描述符，应用程序通过这个描述符进行网络通信。

## 怎么理解

- 可以把套接字理解成“网络通信端点”。
- 一次 TCP 通信至少有两个端点：客户端 socket 和服务器端 socket。
- 服务器端还会区分[[linux网络编程/概念词条/监听套接字|监听套接字]]和[[linux网络编程/概念词条/已连接套接字|已连接套接字]]。

## 常见分类

- 按协议族：IPv4、IPv6、本地套接字等。
- 按传输方式：字节流套接字、数据报套接字。
- 按角色：监听套接字、通信套接字。

## 和文件描述符的关系

- socket 返回值是一个整数文件描述符。
- 可以用 `close` 关闭 socket。
- 对 TCP socket，很多场景也可以用 `read/write` 读写数据。

## 易错点

- `socket` 返回的监听 fd 不直接代表某个客户端连接。
- `accept` 返回的新 fd 才是与具体客户端通信的 fd。
- 套接字不是 IP 地址，也不是端口，而是包含协议、地址、端口等信息的通信端点。
