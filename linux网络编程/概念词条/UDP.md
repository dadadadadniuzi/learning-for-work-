# UDP

## 它是什么

- UDP 是传输层协议，全称 User Datagram Protocol。
- 它提供无连接、面向数据报、开销较小的通信能力。

## 怎么理解

- 无连接：发送前不需要像 TCP 那样先建立连接。
- 面向数据报：每次发送的数据报有天然边界。
- 开销小：协议机制比 TCP 简单，但相应地不保证可靠到达、顺序和不重复。
- 首部简单：UDP 首部固定 8 字节，字段见 [[linux网络编程/概念词条/UDP数据报格式|UDP数据报格式]]。

## 在 socket 中的典型位置

- `socket(AF_INET, SOCK_DGRAM, 0)` 通常创建 UDP 套接字。
- UDP 常配合 [[linux网络编程/函数笔记/Socket/sendto|sendto]]、[[linux网络编程/函数笔记/Socket/recvfrom|recvfrom]] 使用。
- UDP 编程需要重点关注“目标地址”“来源地址”和“数据报边界”，流程见 [[linux网络编程/概念词条/UDP通信流程|UDP通信流程]]。
- UDP 数据报会被封装进 [[linux网络编程/概念词条/IP数据报格式|IP数据报格式]] 的数据部分。

## 和 TCP 的区别

- [[linux网络编程/概念词条/TCP|TCP]] 面向连接，可靠有序，适合文件传输、网页访问等场景。
- UDP 无连接，不保证可靠，适合实时性要求高、可接受少量丢包的场景。
- TCP 的首部和控制机制更复杂，见 [[linux网络编程/概念词条/TCP数据报格式|TCP数据报格式]]；UDP 首部更简单，见 [[linux网络编程/概念词条/UDP数据报格式|UDP数据报格式]]。

## 易错点

- UDP 不等于“不需要协议”，它仍然是传输层协议，只是机制更轻量。
- UDP 不负责重传和顺序保证，如果应用需要这些能力，要自己在应用层设计。
