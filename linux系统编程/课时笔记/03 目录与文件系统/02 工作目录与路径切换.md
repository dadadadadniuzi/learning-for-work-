# 工作目录与路径切换

## 本节学什么

- 当前工作目录
- `getcwd`
- `chdir`
- 相对路径解析




## 本节学什么详解

- 当前工作目录：当前工作目录决定了相对路径从哪里开始解析；切目录、查路径、运行脚本时都要时刻留意它。
- `getcwd`：这一节围绕这个知识点展开，重点记住它解决什么问题、什么时候用、以及和前后内容的关系。
- `chdir`：这一节围绕这个知识点展开，重点记住它解决什么问题、什么时候用、以及和前后内容的关系。
- 相对路径解析：这一节围绕这个知识点展开，重点记住它解决什么问题、什么时候用、以及和前后内容的关系。

## 知识点补充
- 当前工作目录决定了相对路径从哪里开始解释。
- `getcwd` 用来查看当前位置，`chdir` 用来切换当前位置。
- 在写命令行程序或守护进程时，这部分非常重要。

## 本节内容速览

- 相对路径永远依赖当前工作目录
- `getcwd` 用于查看当前位置
- `chdir` 会影响后续所有相对路径操作




## 复习时要回答

- 当前工作目录到底解决什么问题？能不能用自己的话把它讲清楚？
- getcwd和前后内容是什么关系，为什么这一节要把它单独拿出来讲？
- 如果你只允许保留一句话，这一节最该记住的结论是什么？

## 细节补充

- 目录本质上是特殊文件，里面保存的是目录项
- `DIR`、`struct dirent`、`inode` 要连在一起理解
- `.` 和 `..` 是目录遍历里必须主动处理的特殊项
- `getcwd/chdir` 会影响相对路径的解析结果
- 删除、重命名、链接，本质都在改变目录项和 inode 的关系
