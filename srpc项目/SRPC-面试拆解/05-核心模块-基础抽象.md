## 1. `rpc_basic.h`

文件：[rpc_basic.h](G:\计算机学习\srpc项目\srpc-master\src\rpc_basic.h)

作用：公共常量、枚举、工具函数。

### 重点枚举

1. `RPCDataType`
   作用：定义 body 数据格式。
2. `RPCStatusCode`
   作用：统一描述成功和失败原因。
3. `RPCCompressType`
   作用：定义压缩算法。
4. `RPCModuleType`
   作用：定义模块种类。

### 关键函数

1. `RPCCommon::log_format(std::string& key, std::string& value, const RPCLogVector& fields)`
   输入：日志字段列表
   输出：格式化后的 key/value
2. `RPCCommon::addr_to_string(const struct sockaddr *addr, char *ip_str, socklen_t len, unsigned short *port)`
   输入：socket 地址
   输出：IP 字符串、端口
   返回：是否成功

## 3. `rpc_service.h`

文件：[rpc_service.h](G:\计算机学习\srpc项目\srpc-master\src\rpc_service.h)

作用：服务注册和方法分发。

### `class RPCService`

关键函数：

1. `RPCService(const std::string& name)`
   输入：服务名
2. `get_name()`
   输出：服务名
3. `find_method(const std::string& method_name)`
   输入：方法名
   输出：方法函数指针
4. `add_method(const std::string& method_name, rpc_method_t&& method)`
   输入：方法名和对应函数

### `ServiceRPCCallImpl`

作用：把框架层调用桥接到业务函数。

输入：

1. `service`
2. `worker`
3. 成员函数指针

输出：`RPCStatusCode`

## 这一篇要记住什么

1. `RPCService` 管方法映射
2. `RPCContext` 管本次调用上下文
3. `RPCClient` 管客户端配置和任务创建
4. `RPCServer` 管服务注册和请求分发
