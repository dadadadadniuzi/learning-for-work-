# GCC编译流程与链接

## 本节学什么

- 预处理、编译、汇编、链接
- `gcc -E/-S/-c`
- 静态链接与动态链接




## 本节学什么详解

- 预处理、编译、汇编、链接：C 程序从源码到可执行文件要经历四步：预处理展开头文件和宏，编译生成汇编，汇编生成目标文件，链接把多个目标文件和库拼成可执行程序。
- `gcc -E/-S/-c`：这三个参数分别让你停在不同阶段：`-E` 停在预处理，`-S` 停在汇编，`-c` 停在目标文件，适合理解编译流程和排查问题。
- 静态链接与动态链接：静态链接会把库代码直接拷进可执行文件，动态链接则运行时再去装载共享库；前者独立但体积大，后者更省空间但依赖运行环境。

## 知识点补充
- `-E`、`-S`、`-c` 分别对应预处理、编译和汇编。
- 链接阶段会把目标文件和库拼成最终可执行文件。
- 静态链接更独立，动态链接更灵活。
- 像 [[linux系统编程/函数笔记/标准库/putchar|putchar]] 这样的标准库函数，通常在最基础的示例程序里就会出现，适合拿来观察“源码经过编译后怎么变成可执行文件”。

## 内容速览

- 源文件不是直接变成可执行文件的，中间会经过多个阶段。
- 预处理会展开头文件和宏，编译会把 C 变成汇编。
- 汇编会把汇编变成机器码，链接会把目标文件和库拼起来。
- 静态链接更独立，动态链接更灵活。

## 详细说明

### 1. 预处理

- 展开 `#include`
- 替换宏定义
- 处理条件编译
- 生成 `.i` 文件

### 2. 编译

- 把 C 源码翻译成汇编
- 生成 `.s` 文件
- 这一阶段重点是语法和语义检查

### 3. 汇编

- 把汇编翻译成机器码
- 生成 `.o` 目标文件

### 4. 链接

- 把多个目标文件和库文件组合成可执行文件
- 处理符号引用
- 决定函数和全局变量最终来自哪里

## 常用参数

- `gcc -E file.c -o file.i`
- `gcc -S file.i -o file.s`
- `gcc -c file.s -o file.o`
- `gcc file.o -o app`
- `gcc -g`：生成调试信息
- `gcc -I dir`：添加头文件路径
- `gcc -L dir`：添加库路径
- `gcc -lxxx`：链接库 `libxxx.so` 或 `libxxx.a`

## 静态链接与动态链接

- 静态链接：库代码在链接时直接进可执行文件。
- 动态链接：程序运行时再加载共享库。
- 静态链接的优点是运行时依赖少，缺点是文件大。
- 动态链接的优点是节省空间、共享库复用，缺点是运行环境依赖强。
- 如果动态库找不到，程序会在启动阶段就报错。
