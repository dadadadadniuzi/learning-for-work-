# 高并发服务器

## 本章目标

- 理解为什么单进程阻塞式 TCP 服务器一次只能方便地处理一个客户端。
- 掌握[[linux网络编程/概念词条/多进程并发服务器|多进程并发服务器]]的基本模型：父进程负责 `accept`，子进程负责和客户端通信。
- 掌握[[linux网络编程/概念词条/多线程并发服务器|多线程并发服务器]]的基本模型：主线程负责接收连接，工作线程负责通信。
- 理解[[linux网络编程/概念词条/端口复用|端口复用]]为什么能缓解服务器重启时端口暂时不可用的问题。
- 为后续[[linux网络编程/05 IO多路复用|IO 多路复用]]理解“一个执行流管理多个连接”做铺垫。

## 细节补充

- 多进程和多线程并发服务器的共同目标是：让监听连接和处理已连接客户端这两件事不要互相卡住。
- 多进程模型隔离性好，但创建和切换成本更高；多线程模型共享进程资源，通信成本低，但要注意共享数据安全。
- 端口复用通常要在 [[linux网络编程/函数笔记/Socket/bind|bind]] 之前通过 [[linux网络编程/函数笔记/Socket/setsockopt|setsockopt]] 设置，否则 [[linux网络编程/函数笔记/Socket/bind|bind]] 时端口可能仍然不可用。
- 如果想确认某个 socket 是否开启了 [[linux网络编程/概念词条/SO_REUSEADDR|SO_REUSEADDR]] 或 [[linux网络编程/概念词条/SO_REUSEPORT|SO_REUSEPORT]]，可以使用 [[linux网络编程/函数笔记/Socket/getsockopt|getsockopt]] 查询。
