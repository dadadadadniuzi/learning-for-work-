# dup-dup2-fcntl与重定向

## 本节学什么详解

- [[linux系统编程/概念词条/文件描述符|文件描述符]]复制：文件描述符复制解决的是“怎样让多个 fd 指向同一个打开文件对象”这个问题。理解了这一点，你后面才会看懂共享偏移量、重定向和管道端口复用。
- [[linux系统编程/概念词条/标准文件描述符|标准输入输出]]重定向：重定向解决的是“怎样把标准输入输出错误重新接到别的文件或管道上”这个问题。它是 shell 管道、日志输出、守护进程标准模板的基础。
- `dup`：`dup` 用来复制一个文件描述符，并自动选取当前最小可用编号。它适合做最基础的 fd 复制实验，帮助理解“两个 fd 指向同一打开文件对象”。
- `dup2`：`dup2` 用来把旧描述符复制到你指定的新编号上，因此它是实现标准输入输出重定向的主力接口。前面学 `dup` 是为了理解原理，这里学 `dup2` 是为了真正做重定向。
- `fcntl`：`fcntl` 解决的是“怎样更细粒度地控制文件描述符和打开状态”这个问题。它比 `dup/dup2` 更通用，常用于复制描述符、读取/修改标志位、设置非阻塞等。

## 知识点补充
- `dup` 和 `dup2` 的本质是复制文件描述符，让多个 fd 指向同一个打开文件对象。
- 重定向其实就是把标准输入输出错误重新指向别的 fd。
- `fcntl` 负责更细粒度的 fd 控制，常和非阻塞、复制标志一起出现。

## 本节内容速览

- 重定向的本质是让 [[linux系统编程/概念词条/标准文件描述符|标准描述符]] 指向新的打开文件对象
- `dup` 自动找最小可用描述符
- `dup2` 直接复制到指定编号
- `fcntl` 能做更细粒度的描述符控制



## 细节补充

- `open` 的第二个参数也就是 [[linux系统编程/概念词条/open标志位|flags]]，既决定访问方式，也决定是否创建、截断、追加
- `read/write` 不保证一次完整完成，返回值是你处理 IO 的核心依据
- `lseek` 只移动偏移，不会改变文件内容本身
- `stat/lstat` 的差别要和软链接联动理解
- `dup/dup2` 的本质是让多个 [[linux系统编程/概念词条/文件描述符|文件描述符]] 共享同一个打开文件对象
- `fcntl` 是后面非阻塞、描述符控制的常见入口
