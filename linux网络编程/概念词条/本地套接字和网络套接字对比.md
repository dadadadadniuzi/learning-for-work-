# 本地套接字和网络套接字对比

## 它们分别是什么

本地套接字通常指 [[linux网络编程/概念词条/Unix Domain Socket|Unix Domain Socket]]，用于同一台主机内的进程间通信。

网络套接字通常指基于 [[linux网络编程/概念词条/TCP|TCP]] 或 [[linux网络编程/概念词条/UDP|UDP]] 的 socket，用于通过网络协议栈通信，可以同机，也可以跨主机。

## 核心区别一句话

- 本地套接字：同一台机器内通信，用路径或本地地址标识通信端点。
- 网络套接字：通过网络协议栈通信，用 IP + 端口标识通信端点。

## 总体对比

| 对比项      | 本地套接字                                              | 网络套接字                                    |
| -------- | -------------------------------------------------- | ---------------------------------------- |
| 常见名称     | Unix Domain Socket、本地 socket                       | TCP socket、UDP socket、IPv4/IPv6 socket   |
| 通信范围     | 同一台主机内进程                                           | 同一主机或不同主机                                |
| 地址族      | [[AF_UNIX与AF_LOCAL]]                               | `AF_INET` / `AF_INET6`                   |
| 地址结构     | [[sockaddr_un]]                                    | [[sockaddr_in]] / `sockaddr_in6`         |
| 地址标识     | 文件系统路径，常见字段是 [[sun_path]]                          | IP 地址 + 端口                               |
| 是否经过真实网络 | 不经过真实网络                                            | 经过网络协议栈，可跨主机                             |
| 常见类型     | `SOCK_STREAM`、`SOCK_DGRAM`                         | `SOCK_STREAM` 对应 TCP，`SOCK_DGRAM` 对应 UDP |
| 常用流程     | `socket -> bind -> listen -> accept -> read/write` | TCP 类似；UDP 常用 `sendto/recvfrom`          |
| 典型用途     | 本机 IPC、服务进程间通信                                     | 网络服务、客户端服务器通信、跨主机通信                      |

## 代码流程上的相同点

本地套接字和网络套接字都使用 socket API，因此很多函数名相同：

- [[linux网络编程/函数笔记/Socket/socket|socket]]
- [[linux网络编程/函数笔记/Socket/bind|bind]]
- [[linux网络编程/函数笔记/Socket/listen|listen]]
- [[linux网络编程/函数笔记/Socket/accept|accept]]
- [[linux网络编程/函数笔记/Socket/connect|connect]]
- [[linux网络编程/函数笔记/Socket/read|read]]
- [[linux网络编程/函数笔记/Socket/send|send]]
- [[linux网络编程/函数笔记/Socket/recv|recv]]

所以学本地套接字时，不需要重新学习一套完全不同的接口，重点是地址族和地址结构变了。

## 地址结构上的不同

网络套接字 IPv4 常用：

```c
struct sockaddr_in {
    sa_family_t sin_family;  // AF_INET
    in_port_t sin_port;      // 端口
    struct in_addr sin_addr; // IP 地址
};
```

本地套接字常用：

```c
struct sockaddr_un {
    sa_family_t sun_family;  // AF_UNIX / AF_LOCAL
    char sun_path[];         // socket 文件路径
};
```

因此：

- 网络套接字重点关注 IP、端口、字节序转换。
- 本地套接字重点关注路径、socket 文件残留、路径长度。

## 服务器端流程对比

网络 TCP 服务器：

```text
socket(AF_INET, SOCK_STREAM, 0)
memset(&addr, 0, sizeof(addr));
bind(IP + port)
listen
accept
read/write 或 send/recv
```

本地流式 socket 服务器：

```text
socket(AF_UNIX, SOCK_STREAM, 0)
memset(&addr, 0, sizeof(addr));
unlink(path)
bind(path)
listen
accept
read/write 或 send/recv
```

本地套接字服务器常在 [[linux网络编程/函数笔记/Socket/bind|bind]] 前调用 [[linux系统编程/函数笔记/目录与文件系统/unlink|unlink]]，因为上次程序退出后 socket 文件路径可能残留。

## 客户端流程对比

网络 TCP 客户端：

```text
socket(AF_INET, SOCK_STREAM, 0)
connect(server IP + port)
read/write 或 send/recv
```

本地流式 socket 客户端：

```text
socket(AF_UNIX, SOCK_STREAM, 0)
connect(server path)
read/write 或 send/recv
```

二者都可以使用 [[linux网络编程/函数笔记/Socket/connect|connect]]，区别是 connect 的目标地址不同。

## 和 TCP 回环地址的区别

如果两个进程在同一台机器上通信，也可以使用 `127.0.0.1` 或 `localhost` 走 TCP 回环。

但 TCP 回环仍然是网络套接字：

- 地址族是 `AF_INET`。
- 地址是 `127.0.0.1 + 端口`。
- 语义仍然是 TCP/IP 网络通信。

Unix Domain Socket 是本地套接字：

- 地址族是 [[linux网络编程/概念词条/AF_UNIX与AF_LOCAL|AF_UNIX / AF_LOCAL]]。
- 地址常是文件系统路径。
- 更贴近本机 IPC 场景。

## 怎么选择

- 只在同一台主机内通信：优先考虑 [[linux网络编程/概念词条/Unix Domain Socket|Unix Domain Socket]]。
- 未来可能跨主机通信：使用网络套接字。
- 想和浏览器、远程客户端、其他机器通信：使用网络套接字。
- 想让本机多个服务进程用 socket API 通信：本地套接字很合适。

## 易错点

- 本地套接字不能跨主机通信。
- 本地套接字的路径文件残留可能导致 [[linux网络编程/函数笔记/Socket/bind|bind]] 失败，服务器启动前常 [[linux系统编程/函数笔记/目录与文件系统/unlink|unlink]]。
- 本地套接字使用路径，但它不是普通文件读写，通信仍然通过 socket API。
- 网络套接字使用端口，本地套接字通常不使用端口。
- 两者都叫 socket，不代表地址结构可以混用。
