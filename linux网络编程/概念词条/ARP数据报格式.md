# ARP数据报格式

ARP 用来在 IPv4 局域网中根据 IP 地址查询对应的 MAC 地址。它位于网络层和数据链路层之间，常被封装在 [[linux网络编程/概念词条/以太网帧格式|以太网帧]] 中，帧类型为 `0x0806`。

## ARP 解决什么问题

应用程序通常只知道目标 IP，但以太网实际发送帧时需要目标 MAC。ARP 就负责把“目标 IP”解析成“目标 MAC”。

## 整体格式

```text
+----------+----------+------+-----+------+----------------+----------------+
| 硬件类型 | 协议类型 | 硬件长度 | 协议长度 | 操作 | 发送方地址信息 | 目标方地址信息 |
+----------+----------+------+-----+------+----------------+----------------+
```

在以太网 + IPv4 中，地址信息通常展开为：

```text
+----------+----------+-----+-----+------+----------+----------+----------+----------+
| htype    | ptype    | hlen| plen| oper | sha      | spa      | tha      | tpa      |
| 2 字节   | 2 字节   |1字节|1字节|2字节 | 6 字节   | 4 字节   | 6 字节   | 4 字节   |
+----------+----------+-----+-----+------+----------+----------+----------+----------+
```

## 字段说明

| 字段 | 含义 |
|---|---|
| `htype` | 硬件类型，以太网通常为 `1`。 |
| `ptype` | 协议类型，IPv4 通常为 `0x0800`。 |
| `hlen` | 硬件地址长度，以太网 MAC 为 6 字节。 |
| `plen` | 协议地址长度，IPv4 为 4 字节。 |
| `oper` | 操作类型，`1` 表示请求，`2` 表示应答。 |
| `sha` | Sender Hardware Address，发送方 MAC。 |
| `spa` | Sender Protocol Address，发送方 IP。 |
| `tha` | Target Hardware Address，目标 MAC；请求时通常未知。 |
| `tpa` | Target Protocol Address，目标 IP。 |

## ARP 请求与应答

- ARP 请求：广播发送，“谁是这个 IP？请告诉我你的 MAC。”
- ARP 应答：目标主机单播回复，“这个 IP 是我，我的 MAC 是这个。”
- 主机会把解析结果放入 ARP 缓存，避免每次通信都重新广播。

## 和 socket 编程的关系

- 普通 TCP/UDP socket 不需要手写 ARP。
- 当程序要给局域网内或下一跳网关发送 IP 数据报时，内核会自动通过 ARP 获取下一跳 MAC。
- 如果 ARP 解析失败，表现可能是连接超时、无法访问网关或目标主机不可达。

## 易错点

- ARP 解析的是“下一跳 IP 对应的 MAC”，不一定是最终目标主机的 MAC。
- ARP 只用于 IPv4；IPv6 使用邻居发现协议 NDP。
- ARP 请求在局域网内广播，不会跨路由器传播。
