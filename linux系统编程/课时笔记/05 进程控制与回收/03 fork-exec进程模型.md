# fork-exec进程模型

## 本节学什么

- `fork + exec + waitpid`
- 父子进程职责分离
- `system` 的定位




## 本节学什么详解

- `fork + exec + waitpid`：这条组合是典型父子协作模型：`fork` 先复制出子进程，子进程用 `exec` 跑新程序，父进程用 `waitpid` 等待并回收。
- 父子进程职责分离：这一节围绕这个知识点展开，重点记住它解决什么问题、什么时候用、以及和前后内容的关系。
- `system` 的定位：这一节围绕这个知识点展开，重点记住它解决什么问题、什么时候用、以及和前后内容的关系。

## 知识点补充
- 标准做法是 `fork` 后让子进程 `exec` 新程序，父进程 `waitpid` 回收。
- 这是启动子任务、执行外部命令和封装系统工具的基础套路。
- `system` 只是这个模型的便利包装，不是底层主角。

## 本节内容速览

- 父进程负责创建和回收
- 子进程负责切换成目标程序
- `system` 更像方便入口，不是底层主角

## 细节补充

- `wait` 是基础回收，`waitpid` 是更灵活的指定回收
- [[linux系统编程/概念词条/waitpid选项|WNOHANG]] 是非阻塞回收里最常见的选项
- `exec` 成功后不会返回，所以成功路径和失败路径要分开写
- `execl/execlp` 和 `execv/execvp` 的差别在于参数组织方式
- `fork + exec` 是创建新程序的标准套路
