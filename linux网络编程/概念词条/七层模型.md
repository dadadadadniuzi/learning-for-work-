# 七层模型

## 它是什么

- 七层模型通常指 OSI 参考模型，用七个层次描述网络通信过程。
- 它主要用于学习、分析和定位网络问题，帮助我们把复杂通信拆成多个职责明确的部分。

## 七层结构

- 应用层：直接面向用户或应用程序，定义应用数据如何交互。
- 表示层：处理数据格式、编码、压缩、加密等表示问题。
- 会话层：管理通信会话的建立、维持和释放。
- 传输层：负责端到端通信，典型协议有 [[linux网络编程/概念词条/TCP|TCP]] 和 [[linux网络编程/概念词条/UDP|UDP]]。
- 网络层：负责主机到主机的寻址与路由，典型协议是 IP。
- 数据链路层：负责同一链路内的数据帧传输。
- 物理层：负责比特流在物理介质上的传输。

## 怎么理解

- 分层的好处是每层只处理自己的职责，上层不需要知道下层的全部实现细节。
- 发送数据时通常从应用层一路向下封装；接收数据时从物理层一路向上解析。
- 学网络编程时，不需要一开始记住每层所有协议，但要知道 TCP、UDP、IP、大致分别在哪一层。

## 和四层模型的关系

- 七层模型里的应用层、表示层、会话层，在 [[linux网络编程/概念词条/四层模型|四层模型]] 中通常合并为应用层。
- 七层模型里的数据链路层和物理层，在四层模型中通常合并为网络接口层。

## 易错点

- 七层模型不是 Linux socket 编程的直接 API 分类，而是理解网络通信的理论框架。
- 表示层、会话层在实际 TCP/IP 学习中经常不会被单独展开，不代表它们没有意义。
