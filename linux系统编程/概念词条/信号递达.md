# 信号递达

## 它是什么
- 信号从“已经产生”进入到“真正送到进程并开始处理”的阶段。

## 怎么理解
- 递达才意味着信号真的开始生效。
- 如果信号被阻塞，它就会先停在未决状态，等条件允许后再递达。
- 递达之后，要么走默认动作，要么走捕捉函数，要么被忽略。
