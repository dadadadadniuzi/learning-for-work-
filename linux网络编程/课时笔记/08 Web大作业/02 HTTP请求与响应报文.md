## 本节学什么详解

- [[linux网络编程/概念词条/HTTP请求报文|HTTP 请求报文]]：通常由请求行、请求头、空行、请求体组成。
- [[linux网络编程/概念词条/HTTP响应报文|HTTP 响应报文]]：通常由状态行、响应头、空行、响应体组成。
- [[linux网络编程/概念词条/HTTP方法|HTTP 方法]]：例如 `GET`、`POST`，表示这次请求想做什么。
- [[linux网络编程/概念词条/HTTP状态码|HTTP 状态码]]：例如 `200`、`404`、`500`，表示服务器处理结果。
- [[linux网络编程/概念词条/MIME类型|MIME 类型]]：告诉浏览器响应体是什么类型，例如 HTML、CSS、图片。

## 知识点补充

- 课程里的简单 Web 服务器，最常见的是处理 `GET` 请求。
- 如果服务器只会返回文件，最基本也要能拼出状态行、`Content-Type`、空行和文件内容。
- 浏览器是否能正确渲染资源，和响应头里的 `Content-Type` 很有关系。
- 想手工观察请求报文和响应报文，可以直接用 [[linux网络编程/指令查询/命令卡/telnet|telnet]] 连到 Web 服务器端口，再自己输入 HTTP 请求。

## 本节内容速览

- 请求看“请求行 + 请求头”。
- 响应看“状态行 + 响应头 + 响应体”。
- 方法决定请求意图。
- 状态码决定结果语义。
