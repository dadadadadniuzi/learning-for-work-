# MSL

`MSL` 是 Maximum Segment Lifetime 的缩写，意思是“报文最大生存时间”。它表示一个 TCP 报文段在网络中可能存活的最长时间。

## 为什么需要 MSL

网络中的 TCP 报文段不一定会立刻到达，也可能因为路由、重传、排队等原因延迟一段时间。如果旧连接关闭后立刻用相同的四元组建立新连接，旧连接残留的延迟报文就可能干扰新连接。

这里的四元组是：

- 源 IP
- 源端口
- 目的 IP
- 目的端口

## 2MSL 是什么

`2MSL` 就是两个 MSL 时间。主动关闭方进入 [[linux网络编程/概念词条/TIME_WAIT|TIME_WAIT]] 后，需要等待 `2MSL` 才真正回到 `CLOSED`。

等待 `2MSL` 主要解决两个问题：

- 确保最后一个 `ACK` 如果丢失，对端重发 `FIN` 时，主动关闭方还能再次回复 `ACK`。
- 让旧连接中可能滞留在网络里的报文自然过期，避免影响后续相同四元组的新连接。

## 和 TIME_WAIT 的关系

[[linux网络编程/概念词条/TIME_WAIT|TIME_WAIT]] 是 TCP 主动关闭方发送最后一个 `ACK` 后进入的状态。这个状态不会立刻结束，而是等待 `2MSL`。

所以可以这样记：

```text
主动关闭方发送最后 ACK
        ↓
进入 TIME_WAIT
        ↓
等待 2MSL
        ↓
进入 CLOSED
```

## 和四次挥手的关系

在 [[linux网络编程/概念词条/TCP四次挥手|TCP四次挥手]] 中，最后一个 `ACK` 由主动关闭方发送。主动关闭方不能马上销毁连接状态，因为如果这个 `ACK` 丢了，被动关闭方会重发 `FIN`。

主动关闭方保留 `TIME_WAIT` 状态，就能在收到重发的 `FIN` 时再次发送 `ACK`。

## 易错点

- `MSL` 不是某个 socket 函数的参数，而是 TCP 协议层的时间概念。
- `2MSL` 不等于“固定 2 秒”，具体时长由系统实现决定。
- `TIME_WAIT` 多数情况下是正常现象，不应简单理解成程序错误。
- 如果服务端主动关闭大量短连接，服务端也可能出现很多 `TIME_WAIT`。
