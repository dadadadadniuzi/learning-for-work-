# read

> [!info] 功能
> 从[[linux系统编程/概念词条/文件描述符|文件描述符]]读取数据到用户缓冲区。

## 函数原型

- [[linux系统编程/概念词条/ssize_t|ssize_t]] read(int fd, void \*buf, [[linux系统编程/概念词条/size_t|size_t]] count);

## 依赖头文件

- `#include <unistd.h>`

## 输入参数

- `fd`：要读取的[[linux系统编程/概念词条/文件描述符|文件描述符]]。
- `buf`：接收数据的缓冲区地址，类型是 `void \*`，表示 `read` 会把读到的数据写入这块内存，所以这块内存必须可写、容量足够。
- `count`：期望读取的最大字节数，类型是 [[linux系统编程/概念词条/size_t|size_t]]。

## 输出参数

- `buf`：函数成功后，缓冲区前面一部分会被新读入的数据覆盖。

## 返回值

- 成功返回实际读取的字节数，类型是 [[linux系统编程/概念词条/ssize_t|ssize_t]]。
- 返回 `0` 表示读到了文件结尾 `EOF`，或者对端已经关闭。
- 失败返回 `-1`，并设置 [[linux系统编程/概念词条/errno|errno]]。

## 知识点补充

- `read` 不保证一次就读满 `count` 字节，出现短读是正常现象。
- 对普通文件、管道、终端、套接字，`read` 的阻塞行为和返回时机并不完全一样。
- 如果读的是文本数据，`read` 不会自动补字符串结尾的 `\0`，这一点和高级语言字符串读取很不一样。

## 常见用法

- 从文件、管道或标准输入循环读取数据。
- 和 [[linux系统编程/函数笔记/文件IO/write.md|write]] 配合实现拷贝程序。

## 易错点

- 不要把返回值当成恒等于 `count`。
- 读出来的二进制数据不能默认当 C 字符串使用。
- 失败时通常要结合 [[linux系统编程/概念词条/perror|perror]] 或 [[linux系统编程/概念词条/errno|errno]] 看错误原因。
