# strspn

> [!info] 功能
> 计算字符串开头连续有多少个字符都属于指定字符集合。

## 函数原型

- `size_t strspn(const char *s, const char *accept);`

## 依赖头文件

- `#include <string.h>`

## 输入参数

- `s`
  要被扫描的源字符串。
  它必须是一个合法的、以 `\0` 结尾的 C 风格字符串。

- `accept`
  允许出现的字符集合。
  `strspn` 会从 `s` 的开头开始检查，只要当前字符还属于这个集合，就继续向后统计。

例如：

- `accept = "0123456789"`：统计开头连续数字的长度
- `accept = " \t\n"`：统计开头连续空白字符的长度
- `accept = "abc"`：统计开头连续由 `a`、`b`、`c` 组成的长度

## 输出参数

- 无直接输出参数。

## 返回值

- 返回从 `s` 开头开始，连续属于 `accept` 集合的字符个数。
- 返回值类型是 `size_t`。

## 怎么理解

`strspn` 只看“开头这一段”。

它不会去找整个字符串里所有命中的字符，而是：

1. 从第一个字符开始检查
2. 只要字符还属于 `accept`，就继续计数
3. 一旦遇到不属于 `accept` 的字符，就立刻停止

例如：

```c
size_t n = strspn("123abc", "0123456789");
```

这里结果是 `3`，因为开头连续的 `1`、`2`、`3` 都是数字，到了 `a` 就停止。

## 常见用法

```c
#include <stdio.h>
#include <string.h>

int main(void) {
    size_t n = strspn("   hello", " \t\n");
    printf("%zu\n", n);
    return 0;
}
```

这里会输出 `3`，表示开头有 3 个连续空格。

## 典型场景

- 跳过字符串开头的空白字符
- 识别开头连续数字
- 做简单词法分析时切分前缀
- 判断一个前缀是否全部由某类字符组成

## 易错点

- `strspn` 返回的是长度，不是指针。
- 它只检查开头连续的一段，不会扫描整个字符串寻找所有命中字符。
- `accept` 是字符集合，不是一个必须按顺序匹配的子串。
- 返回 `0` 并不表示出错，只可能表示第一个字符就不在集合里。

## 和相关函数的区别

- `strspn`：统计开头连续属于某集合的长度。
- `strpbrk`：找第一个属于某集合的字符位置。
- `strcpy`：复制字符串。
- `strcat`：拼接字符串。

## 知识点补充

- `strspn` 的名字可以理解成 string span。
- 它常和“跳过前导空白”“识别数字前缀”这类处理一起出现。
- 如果你已经拿到了返回值 `n`，那么 `s + n` 往往就是“第一个不满足条件的位置”。
