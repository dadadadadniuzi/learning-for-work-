# 端口复用

端口复用是通过 socket 选项让程序在特定条件下复用本地 IP 和端口。网络编程课程中最常见的用途是：服务器重启时，避免因为旧连接残留在 [[linux网络编程/概念词条/TIME_WAIT|TIME_WAIT]] 等状态而导致 [[linux网络编程/函数笔记/Socket/bind|bind]] 失败。

## 为什么需要端口复用

TCP 服务器退出后，连接状态不会总是立刻完全消失。尤其是主动关闭连接的一方，可能进入 [[linux网络编程/概念词条/TIME_WAIT|TIME_WAIT]]，等待 [[linux网络编程/概念词条/MSL|2MSL]] 后才完全释放。

如果服务端马上重启并绑定同一个端口，可能遇到：

```text
bind: Address already in use
```

端口复用就是用来缓解这类“端口短时间不可重新绑定”的问题。

## 常用相关函数

- [[linux网络编程/函数笔记/Socket/setsockopt|setsockopt]]：设置 socket 选项，端口复用主要靠它开启。
- [[linux网络编程/函数笔记/Socket/getsockopt|getsockopt]]：查询 socket 选项，调试时可用于确认选项是否已设置。
- [[linux网络编程/函数笔记/Socket/bind|bind]]：绑定本地 IP 和端口，端口复用选项必须在它之前设置。

## 常用相关选项

- [[linux网络编程/概念词条/SOL_SOCKET|SOL_SOCKET]]：表示设置的是通用 socket 层选项。
- [[linux网络编程/概念词条/SO_REUSEADDR|SO_REUSEADDR]]：最常见的地址复用选项，常用于服务器快速重启。
- [[linux网络编程/概念词条/SO_REUSEPORT|SO_REUSEPORT]]：允许多个 socket 绑定同一 IP 和端口，常用于多进程/多线程负载分担。

## 典型写法

```c
int lfd = socket(AF_INET, SOCK_STREAM, 0);

int opt = 1;
setsockopt(lfd, SOL_SOCKET, SO_REUSEADDR, &opt, sizeof(opt));

bind(lfd, (struct sockaddr *)&servaddr, sizeof(servaddr));
listen(lfd, 128);
```

## 调用顺序

```text
socket
  ↓
setsockopt 设置 SO_REUSEADDR / SO_REUSEPORT
  ↓
bind
  ↓
listen
```

最关键的一点：端口复用通常必须在 [[linux网络编程/函数笔记/Socket/bind|bind]] 之前设置。

## SO_REUSEADDR 和 SO_REUSEPORT 的区别

| 选项 | 主要用途 | 典型场景 |
|---|---|---|
| [[linux网络编程/概念词条/SO_REUSEADDR|SO_REUSEADDR]] | 允许复用本地地址，缓解端口处于等待状态导致的重启失败 | 调试服务器、服务快速重启 |
| [[linux网络编程/概念词条/SO_REUSEPORT|SO_REUSEPORT]] | 允许多个 socket 绑定同一 IP 和端口，并由内核分配连接或报文 | 多进程服务器负载均衡 |

## 它不能解决什么

- 不能随便抢占另一个正在正常监听的服务端口。
- 不能让两个没有设置正确选项的 TCP 服务器同时监听同一四元组。
- 不能替代关闭无用进程；真正被进程占用的端口仍要用 [[linux网络编程/指令查询/命令卡/ss|ss]]、[[linux网络编程/指令查询/命令卡/netstat|netstat]]、[[linux网络编程/指令查询/命令卡/lsof|lsof]] 排查。

## 排查命令

```bash
ss -tulpn | grep 9527
netstat -tulpn | grep 9527
lsof -i :9527
```

## 易错点

- 忘记在 `bind` 前设置，端口复用不会对这次绑定生效。
- `optval` 要传地址，例如 `&opt`，不是直接传整数 `1`。
- `SO_REUSEADDR` 和 `SO_REUSEPORT` 不是同一个东西。
- 端口复用不是“安全地忽略所有端口冲突”，它只是放宽某些绑定限制。
