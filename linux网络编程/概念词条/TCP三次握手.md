# TCP三次握手

TCP 三次握手是 [[linux网络编程/概念词条/TCP|TCP]] 建立连接时客户端和服务器之间交换控制报文的过程。它发生在应用层代码调用 [[linux网络编程/函数笔记/Socket/connect|connect]] 和 [[linux网络编程/函数笔记/Socket/accept|accept]] 的背后。

## 为什么需要三次握手

- 确认客户端的发送能力和接收能力正常。
- 确认服务器的发送能力和接收能力正常。
- 双方协商初始序号，后续 TCP 才能用序号、确认号保证可靠传输和有序传输。
- 避免历史失效连接请求突然到达服务器，导致服务器错误建立连接。

## 三次过程

| 次数 | 方向 | 报文标志 | 含义 |
|---|---|---|---|
| 第一次 | 客户端 -> 服务器 | `SYN` | 客户端请求建立连接，并发送自己的初始序号 `seq=x`。 |
| 第二次 | 服务器 -> 客户端 | `SYN + ACK` | 服务器同意建立连接，确认客户端序号 `ack=x+1`，同时发送自己的初始序号 `seq=y`。 |
| 第三次 | 客户端 -> 服务器 | `ACK` | 客户端确认服务器序号 `ack=y+1`，连接正式建立。 |

## 和 socket 函数的关系

- 客户端调用 [[linux网络编程/函数笔记/Socket/connect|connect]] 时，会触发三次握手。
- 服务器先调用 [[linux网络编程/函数笔记/Socket/listen|listen]] 进入监听状态。
- 服务器调用 [[linux网络编程/函数笔记/Socket/accept|accept]] 等待已经完成握手的连接，成功后返回 [[linux网络编程/概念词条/已连接套接字|已连接套接字]]。

## 状态变化速记

- 客户端：`CLOSED -> SYN_SENT -> ESTABLISHED`。
- 服务器：`LISTEN -> SYN_RCVD -> ESTABLISHED`。

## 图示

```text
客户端                                      服务器
  | -------- SYN, seq=x -----------------> |
  | <--- SYN+ACK, seq=y, ack=x+1 --------- |
  | -------- ACK, ack=y+1 ---------------> |
  |              连接建立                  |
```

## 易错点

- 三次握手是 TCP 协议栈完成的，不是应用程序自己手动发送 `SYN`、`ACK`。
- `connect` 成功通常表示三次握手已经完成。
- `accept` 返回的新 fd 才是和客户端通信的 fd，监听 fd 继续负责接收新连接。
