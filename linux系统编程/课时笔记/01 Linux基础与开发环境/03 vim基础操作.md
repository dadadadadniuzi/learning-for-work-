# vim基础操作

## 本节学什么

- 命令模式、插入模式、末行模式
- 光标移动、复制粘贴、删除、撤销
- 保存退出、替换、分屏




## 本节学什么详解

- 命令模式、插入模式、末行模式：vim 的核心是模式切换：命令模式负责移动和操作，插入模式负责输入正文，末行模式负责保存、查找、替换和执行命令。
- 光标移动、复制粘贴、删除、撤销：这一节围绕这个知识点展开，重点记住它解决什么问题、什么时候用、以及和前后内容的关系。
- 保存退出、替换、分屏：这一节不是只学“怎么退出 vim”，而是要把保存、退出、批量替换、窗口分屏这些高频操作串成一个完整工作流。

## 知识点补充
- `vim` 的核心是模式切换，不同模式负责编辑、命令和选择。
- 先学会打开、保存、退出，再学会移动、删除、复制和搜索。
- 练习时重点是把常见操作变成肌肉记忆。

## 本节内容速览

- `ESC` 回到命令模式
- `i/a/o` 是进入插入模式的高频入口
- 末行模式负责保存、搜索、替换、显示行号
- `vim` 熟练后，写和改 C 程序会更顺手



## 细节补充

- 这部分的重点不是背命令，而是记住命令对应的文件系统行为
- `/proc`、`/dev`、`/etc`、`/home`、`/root` 这几个目录在课程里经常重复出现
- `Tab` 补全、历史命令、光标移动、删除字符，都会直接影响后续编程效率
- `gcc` 的四步编译要能和产生的文件后缀对应起来
- `gdb` 是后面调试进程、线程、信号问题的基础工具
