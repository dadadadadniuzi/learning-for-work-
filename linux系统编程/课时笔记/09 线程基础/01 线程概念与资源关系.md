# 线程概念与资源关系

## 本节学什么

- 线程与进程的区别
- 线程共享资源与独享资源
- 线程 ID 的意义




## 本节学什么详解

- 线程与进程的区别：进程是资源分配的基本单位，线程是 CPU 调度的基本单位；线程共享进程资源，但各自有独立的执行流。
- 线程共享资源与独享资源：线程之间共享地址空间、文件描述符等资源，但每个线程仍有自己的栈、寄存器上下文和调度状态。
- 进程：有独立的 进程地址空间。有独立的pcb。 分配资源的最小单位。
	线程：有独立的pcb。没有独立的进程地址空间。 最小单位的执行。
- 线程 ID 的意义：线程 ID 是线程的身份标识，主要用于识别当前执行的是哪个线程，以及在调试、同步和取消时定位目标。
- linux的线程本质上是轻量级进程

## 知识点补充
- 线程是进程内的执行流，多个线程共享同一进程资源。
- 每个线程有自己的栈和寄存器上下文。
- 理解“共享与私有”是线程编程的基础。
- ps -Lf 进程id 查看指定线程的线程号---> 线程号 LWP
- 线程中检查出错返回：  （线程一般不设errno）
	fprintf(stderr, "xxx error: %s\n", strerror(ret));
- 线程共享：
	独享 栈空间（内核栈、用户栈）
	共享 ./text./data ./rodataa ./bsss heap  ---> 共享【全局变量】（errno）
	子线程里更改全局变量后，主线程里也跟着发生变化

## 本节内容速览

- 线程共享代码段、全局数据、堆
- 每个线程有自己的栈
- 多线程真正的难点在共享资源访问

## 细节补充

- 线程共享地址空间、堆、全局变量，私有的是栈和线程上下文
- `pthread_create` 创建新线程后，入口函数会立即开始执行
- `pthread_join` 用于回收线程并获取线程返回值
- `pthread_detach` 表示线程结束后自动回收资源
- `pthread_cancel` 是取消请求，不是简单的强制杀死
