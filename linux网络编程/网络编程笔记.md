## 课件整理版

### 一条主线

网络编程的核心目标是让两个进程稳定交换数据，而实现路径逐步升级：

- 先会写单连接 TCP 程序
- 再支持多个客户端同时接入
- 再提升并发能力和事件处理效率
- 最后过渡到事件驱动框架

### 核心 API 速记

| 主题 | 核心函数 | 必会点 |
| --- | --- | --- |
| 地址转换 | `inet_pton`, `inet_ntop` | 文本地址与网络地址互转 |
| TCP 服务端 | `socket`, `bind`, `listen`, `accept` | 建立监听流程 |
| TCP 客户端 | `socket`, `connect` | 发起连接 |
| 数据收发 | `send`, `recv`, `read`, `write` | 返回值与连接状态 |
| UDP | `sendto`, `recvfrom` | 无连接通信 |
| 多路复用 | `select`, `poll`, `epoll_wait` | 同时管理多连接 |

### 易错点

- 没有处理短读短写
- 把 TCP 当成“天然一条消息一次发送一次接收”
- `select` 集合每次循环前忘记重置
- `epoll` ET 模式下没有一次性读到 `EAGAIN`
- 忽略客户端异常断开和资源清理
