## 本节学什么详解

- `fork`：这里的 `fork` 解决的是“怎样先让父进程退出、把后续守护化动作交给子进程继续做”这个问题。它的作用已经不只是创建子进程，而是为后面的 `setsid` 创造条件。
- `setsid`：这里的 `setsid` 解决的是“怎样让进程脱离原来的会话和控制终端”这个问题。没有这一步，进程仍可能受到终端关闭、前后台控制等影响。
- `chdir`：这里的 `chdir` 解决的是“怎样避免守护进程长期占住某个挂载目录”这个问题。守护进程通常会切到根目录或安全目录，避免阻止文件系统卸载。
- `umask`：`umask` 决定新建文件/目录的默认权限会被屏蔽掉哪些位；它解释了“为什么我创建出来的权限和预期不一样”。
- 重定向标准输入输出：这一部分解决的是“怎样让守护进程彻底摆脱终端的标准输入、标准输出和标准错误”这个问题。它把前面学过的 `dup2`、标准描述符等知识真正用到了模板里。
- [[linux系统编程/指令查询/umask|umask]]：`umask` 决定新建文件/目录的默认权限会被屏蔽掉哪些位；它解释了“为什么我创建出来的权限和预期不一样”。

## 知识点补充
- 守护进程的目标是脱离终端、后台稳定运行。
- 标准模板通常是 `fork -> setsid -> chdir -> umask -> 重定向`。
- 关闭或重定向标准输入输出错误，是避免占用终端的关键。
- 守护进程：
	daemon进程。通常运行于操作系统后台，脱离控制终端。一般不与用户直接交互。周期性的等待某个事件发生或周期性执行某一动作。
	不受用户登录注销影响。通常采用以d结尾的命名方式。
	创建守护进程，最关键的一步是调用setsid函数创建一个新的Session，并成为Session Leader
- 守护进程创建步骤：
	1. fork子进程，让父进程终止。
	2. 子进程调用 setsid() 创建新会话
	3. 通常根据需要，改变工作目录位置 chdir()， 防止目录被卸载。
	4. 通常根据需要，重设umask文件权限掩码，影响新文件的创建权限。  022 -- 755 0345 --- 432   r---wx-w-   422
	5. 通常根据需要，关闭/重定向 文件描述符
	6. 守护进程 业务逻辑。while（）

## 本节内容速览

- 标准套路是：`fork -> 父进程退出 -> setsid -> chdir -> [[linux系统编程/指令查询/umask|umask]] -> 关闭/重定向描述符`
- 目的都是为了脱离终端、稳定运行

## 细节补充

- 守护进程的目标是脱离终端并长期稳定运行
- `setsid`、`chdir`、[[linux系统编程/指令查询/umask|umask]]、重定向是标准模板
- 组长进程不能直接创建会话，所以常先 `fork`
- 标准输入输出错误常被重定向到 `/dev/null`
- 这部分内容后面会直接影响服务器程序设计
