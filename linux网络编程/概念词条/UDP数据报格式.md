# UDP数据报格式

UDP 数据报是 [[linux网络编程/概念词条/UDP|UDP]] 的传输单位。UDP 首部非常简单，固定 8 字节，因此开销小，但它不提供 TCP 那样的可靠传输、顺序保证和连接管理。

## 整体格式

```text
UDP 数据报 = UDP 首部 + UDP 数据
```

```text
+----------+----------+----------+----------+----------------+
| 源端口   | 目的端口 | 长度     | 校验和   | 数据           |
| 2 字节   | 2 字节   | 2 字节   | 2 字节   | 可变长度       |
+----------+----------+----------+----------+----------------+
```

## 字段说明

| 字段 | 长度 | 作用 |
|---|---:|---|
| 源端口 | 2 字节 | 发送方应用进程端口，可用于对端回复。 |
| 目的端口 | 2 字节 | 接收方应用进程端口。 |
| 长度 | 2 字节 | UDP 首部加 UDP 数据的总长度。 |
| 校验和 | 2 字节 | 用于检测 UDP 首部和数据是否出错。 |
| 数据 | 可变 | 应用层数据。 |

## UDP 的“数据报边界”

UDP 是面向数据报的协议。一次发送对应一个 UDP 数据报，接收方一次 `recvfrom` 通常按数据报为单位接收。

这和 [[linux网络编程/概念词条/TCP|TCP]] 不同：TCP 是字节流协议，不保留应用层消息边界。

## 和 IP 的关系

UDP 数据报会被放进 [[linux网络编程/概念词条/IP数据报格式|IP 数据报]] 的数据部分。IP 首部中的 `Protocol` 字段为 `17` 时，表示上层协议是 UDP。

## 和 socket 编程的关系

- `socket(AF_INET, SOCK_DGRAM, 0)` 通常创建 UDP socket。
- UDP 常用 `sendto` / `recvfrom`，因为每个数据报通常都要带目标地址或来源地址。
- 普通应用程序不需要手写 UDP 首部，内核会自动封装。

## 易错点

- UDP 有数据边界，但不保证可靠到达。
- UDP 不建立连接，没有三次握手和四次挥手。
- UDP 首部里的长度是 UDP 首部 + 数据，不是整个以太网帧或 IP 数据报长度。
