## 2. `rpc_message_srpc.h/.cc`

文件：

- [rpc_message_srpc.h](G:\计算机学习\srpc项目\srpc-master\src\message\rpc_message_srpc.h)
- [rpc_message_srpc.cc](G:\计算机学习\srpc项目\srpc-master\src\message\rpc_message_srpc.cc)

### `SRPCMessage`

这是 SRPC 协议的核心消息类。

关键函数：

1. `encode(...)`
   输入：iov 数组、最大数量、size limit
   输出：使用的 iovec 数量或错误
2. `append(...)`
   输入：收到的数据块
   输出：是否收完整包
3. `serialize_meta()`
4. `deserialize_meta()`
5. `serialize(const ProtobufIDLMessage *pb_msg)`
6. `deserialize(ProtobufIDLMessage *pb_msg)`
7. `serialize(const ThriftIDLMessage *thrift_msg)`
8. `deserialize(ThriftIDLMessage *thrift_msg)`
9. `compress()`
10. `decompress()`

### `SRPCRequest` / `SRPCResponse`

负责：

1. service name
2. method name
3. status code
4. error
5. errmsg

### `SRPCHttpRequest` / `SRPCHttpResponse`

核心思路：

1. URL = `/{service}/{method}`
2. `Content-Type` 表示 body 格式
3. `Content-Encoding` 表示压缩
4. `SRPC-Status` 和 `SRPC-Error` 表示 RPC 状态

## 4. `rpc_task.inl`

文件：[rpc_task.inl](G:\计算机学习\srpc项目\srpc-master\src\rpc_task.inl)

这是整个框架最重要的底层实现之一。

### `RPCWorker`

作用：把 `ctx + req + resp + 输入输出对象` 绑在一起。

关键函数：

1. `set_server_input(...)`
2. `set_server_output(...)`
3. `server_serialize()`

### `RPCClientTask`

关键函数：

1. `serialize_input(...)`
2. `check_request()`
3. `message_out()`
4. `finish_once()`
5. `rpc_callback(...)`
6. `log(...)`
7. `add_baggage(...)`
8. `set_http_header(...)`

### `RPCServerTask`

关键函数：

1. `message_out()`
2. `handle(int state, int error)`
3. `get_remote(...)`

### 这一层为什么重要

因为 RPC 在这里真正变成了 Workflow 的任务：

1. 可异步调度
2. 可加入 series
3. 可与其他 Workflow task 串联
