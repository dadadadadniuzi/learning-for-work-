# fork创建子进程

## 本节学什么

- `fork`
- 父子进程返回值差异
- 写时拷贝




## 本节学什么详解

- `fork`：这一节围绕这个知识点展开，重点记住它解决什么问题、什么时候用、以及和前后内容的关系。
- 父子进程返回值差异：`fork` 之后父子进程会得到不同的返回值：父进程拿到子进程 PID，子进程拿到 0，失败时返回 -1。这个差异正是区分执行分支的关键。
- 写时拷贝：这一节围绕这个知识点展开，重点记住它解决什么问题、什么时候用、以及和前后内容的关系。

## 知识点补充
- `fork` 会复制当前进程，父子进程从同一位置继续执行。
- 它们返回值不同，因此可以用返回值区分父进程和子进程。
- 现代 Linux 中常结合写时复制理解它的效率。

## 本节内容速览

- `fork` 成功后父子进程从同一位置继续执行
- 父进程拿到子 PID，子进程拿到 0
- 父子逻辑通常靠返回值分支

## 细节补充

- 进程不是程序本身，而是程序运行后形成的实体
- PCB 记录了进程调度、资源和状态，是理解进程的关键
- `fork` 后父子进程从同一位置继续执行，但返回值不同
- 写时拷贝让父子进程先共享物理页面，直到发生修改
- `exit` 和 `_exit` 的区别在 fork 场景里尤其重要
