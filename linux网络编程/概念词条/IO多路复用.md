# IO多路复用

## 它是什么

- IO 多路复用是一种让一个进程或线程同时监听多个 fd 的技术。
- 它让内核帮应用程序等待事件，应用程序只处理已经就绪的 fd。

## 解决什么问题

- 单线程阻塞式服务器如果卡在某个 `accept` 或 `recv`，就难以及时处理其他连接。
- IO 多路复用让程序可以同时关注监听 socket 和多个通信 socket。

## 易错点

- IO 多路复用本身不负责读写数据，只负责告诉你哪些 fd 可以读写。
- 真正的数据处理仍然需要应用程序调用 `accept`、`recv`、`send` 等函数。
