# 进程组

## 它是什么

- 进程组是把一组相关进程组织在一起的机制。
- 同一个进程组里的进程共享一个进程组 ID，通常记作 PGID。

## 为什么会有它

- 操作系统需要按“组”来管理一批相关进程，例如 shell 启动的一条前台命令链。
- 信号投递、前后台作业控制、终端控制，经常都是以进程组为单位进行的。

## 怎么理解

- 单个进程是一条执行线。
- 多个相关进程可以组成一个[[linux系统编程/概念词条/进程组|进程组]]。
- 多个进程组再归属于同一个会话。

## 学习提示

- 不要把“进程号 PID”“进程组号 PGID”“会话 ID SID”混在一起。
- `setsid` 成功后，新进程会成为新会话首进程，同时也成为新进程组的组长。
