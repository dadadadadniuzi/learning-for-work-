# 事件就绪

## 它是什么

- 事件就绪表示某个 fd 当前满足某种 IO 条件，例如可以读、可以写或发生异常。

## 常见含义

- 监听 socket 可读：通常表示有新连接可以 `accept`。
- 通信 socket 可读：通常表示有数据可读，或对端关闭连接。
- socket 可写：通常表示发送缓冲区有空间。

## 怎么理解

- IO 多路复用返回的是“可以做某件事”的提示。
- 应用程序收到提示后，还要调用具体函数处理，例如 `accept` 或 `recv`。

## 易错点

- 可读不一定代表一定有正常数据，也可能是对端关闭。
- 可写不代表对端已经接收，只代表本地发送条件满足。
