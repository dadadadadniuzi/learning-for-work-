# 路径绑定

## 它是什么

- 路径绑定指 Unix Domain Socket 服务端把 socket 绑定到某个本地文件系统路径。

## 怎么理解

- IPv4 服务端通过 IP 和端口被客户端找到。
- Unix Domain Socket 服务端通过 `sun_path` 指定的路径被客户端找到。
- 客户端使用相同路径调用 `connect`，就能连接服务端。

## 常见流程

- 服务端准备 `sockaddr_un`。
- 设置 `sun_family = AF_UNIX`。
- 设置 `sun_path`。
- 调用 `unlink(path)` 删除旧路径。
- 调用 `bind` 绑定路径。

## 易错点

- 路径已经存在时，`bind` 可能失败。
- socket 路径不是普通数据文件，不要按普通文件读写它。
