## 本节学什么详解

- [[linux网络编程/概念词条/浏览器访问服务器流程|浏览器访问服务器流程]]：浏览器输入 URL 后，会解析目标地址、建立连接、发送请求、接收响应、解析 HTML，再继续请求页面里引用的其他资源。
- URL 到资源返回：URL 指出目标主机和路径，服务器根据路径找到资源并返回响应。
- 多资源请求：HTML 往往只是入口，页面里的 CSS、JS、图片会触发更多请求。
- [[linux网络编程/概念词条/静态资源|静态资源]]：内容不需要服务器动态生成，通常直接从文件系统读取返回。

## 知识点补充

- 浏览器和服务器之间常常不是“一来一回”就结束，而是多轮 HTTP 请求。
- 简单课程项目里，通常只实现“静态资源返回”，也就是把磁盘上的文件映射成 HTTP 响应返回给浏览器。
- 如果路径对应资源不存在，服务器应返回 `404 Not Found`。

## 本节内容速览

- 浏览器输入 URL。
- 浏览器建立 TCP 连接。
- 发送 HTTP 请求。
- 服务器返回 HTTP 响应。
- 浏览器解析页面并继续请求依赖资源。
