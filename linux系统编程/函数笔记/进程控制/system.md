# system

> [!info] 功能
> 通过 shell 执行命令字符串。

## 函数原型

- `int system(const char *command);`

## 依赖头文件

- `#include <stdlib.h>`

## 输入参数

- `command`：传给 shell 的命令字符串，通常会交给 `/bin/sh -c` 处理。传 `NULL` 时可用于检查 shell 是否可用。

## 输出参数

- 无直接输出参数。

## 返回值

- 成功时返回命令终止状态的编码。
- 失败返回 `-1`。

## 知识点补充

- `system` 的内部思路通常可以理解成 [[linux系统编程/函数笔记/进程控制/fork.md|fork]] + `exec` + [[linux系统编程/函数笔记/进程控制/wait.md|wait]] 的组合。
- 命令字符串会经过 shell 解释，所以像重定向、管道、通配符等 shell 语法都可以使用。
- 它方便但不够轻量，也不适合在信号处理函数里使用。

## 常见用法

- 临时执行一条 shell 命令。
- 快速调用外部工具做脚本式操作。

## 易错点

- 命令字符串如果来自不可信输入，容易引入 shell 注入风险。
- 不要在高频或高性能路径里滥用 `system`。
