# IP数据报格式

IP 数据报是网络层的传输单位。它负责把上层的 [[linux网络编程/概念词条/TCP数据报格式|TCP 报文段]] 或 [[linux网络编程/概念词条/UDP数据报格式|UDP 数据报]] 从源主机送到目标主机。

## 所属层次

- OSI：网络层。
- TCP/IP 四层模型：网络层。
- 主要解决的问题：跨网络寻址和路由转发。

## 整体格式

```text
IP 数据报 = IP 首部 + 数据部分
```

IPv4 首部最小 20 字节，带选项时更长。

## IPv4 首部字段

| 字段 | 作用 |
|---|---|
| 版本 Version | IPv4 值为 4。 |
| 首部长度 IHL | 表示 IP 首部长度。 |
| 服务类型 TOS/DSCP | 描述服务质量需求，入门阶段了解即可。 |
| 总长度 Total Length | 整个 IP 数据报长度，包括首部和数据。 |
| 标识 Identification | 分片重组时使用。 |
| 标志 Flags | 控制是否允许分片、是否还有后续分片。 |
| 片偏移 Fragment Offset | 分片在原始数据报中的位置。 |
| 生存时间 TTL | 每经过一个路由器通常减 1，减到 0 被丢弃，防止环路。 |
| 协议 Protocol | 指出数据部分是什么协议，如 TCP=6、UDP=17、ICMP=1。 |
| 首部校验和 Header Checksum | 检查 IP 首部是否出错。 |
| 源 IP 地址 | 发送方 IP。 |
| 目的 IP 地址 | 接收方 IP。 |
| 选项 Options | 可选字段，普通应用编程很少直接关注。 |
| 数据 Payload | 上层数据，如 TCP 或 UDP。 |

## Protocol 字段常见值

- `6`：[[linux网络编程/概念词条/TCP|TCP]]，数据部分是 [[linux网络编程/概念词条/TCP数据报格式|TCP数据报格式]]。
- `17`：[[linux网络编程/概念词条/UDP|UDP]]，数据部分是 [[linux网络编程/概念词条/UDP数据报格式|UDP数据报格式]]。
- `1`：ICMP，常见于 [[linux网络编程/指令查询/命令卡/ping|ping]]。

## 和以太网帧的关系

IP 数据报通常被放进 [[linux网络编程/概念词条/以太网帧格式|以太网帧]] 的数据部分。以太网帧负责同一链路上的 MAC 投递，IP 负责跨网络的路由投递。

## 和 socket 编程的关系

- 调用 [[linux网络编程/函数笔记/Socket/connect|connect]] 时使用的服务器 IP，会成为 IP 层目的地址的一部分。
- 调用 [[linux网络编程/函数笔记/Socket/send|send]] / [[linux网络编程/函数笔记/Socket/recv|recv]] 时，普通应用程序通常只处理 TCP/UDP 数据，不手写 IP 首部。
- 若要观察 IP 首部字段，可使用 [[linux网络编程/指令查询/命令卡/tcpdump|tcpdump]] 抓包。

## 易错点

- IP 只提供尽力而为投递，不保证可靠到达；可靠性通常由 TCP 或应用层实现。
- IP 地址负责主机寻址，端口负责进程寻址。
- IP 分片发生在网络层，TCP 的粘包/半包是传输层字节流和应用层协议边界问题，不要混在一起。
