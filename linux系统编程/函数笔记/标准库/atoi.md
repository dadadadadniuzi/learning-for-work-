# atoi

> [!info] 功能
> 把字符串转换成 `int` 整数，名字来自 ASCII to integer。

## 函数原型

- `int atoi(const char *nptr);`

## 依赖头文件

- `#include <stdlib.h>`

## 输入参数

- `nptr`
  指向要转换的 C 风格字符串。
  这个字符串通常可以包含：
  - 前导空白字符，比如空格、`\t`、`\n`
  - 可选的正负号 `+` / `-`
  - 后续连续的十进制数字字符

常见例子：

- `"123"`
- `"-45"`
- `"   78"`
- `"+1024"`

## 输出参数

- 无输出参数。

## 返回值

- 成功时返回转换后的 `int` 值。
- 如果字符串不能形成有效数字，通常返回 `0`。

## 怎么理解

`atoi` 常用于把“文本形式的数字”快速变成“程序里可计算的整数”。

典型场景：

- 读取命令行参数中的端口号、线程数、循环次数
- 读取配置文件中的数字字符串
- 处理从标准输入读到的整数字符串

示例：

```c
int port = atoi("8080");
int num = atoi("-12");
```

## 常见用法

```c
int main(int argc, char *argv[]) {
    if (argc < 2) {
        return 1;
    }

    int port = atoi(argv[1]);
    return 0;
}
```

如果执行：

```bash
./app 9527
```

那么 `argv[1]` 是字符串 `"9527"`，`atoi(argv[1])` 返回整数 `9527`。

## 易错点

- `atoi` 无法区分“字符串本身就是 `0`”和“转换失败后返回 `0`”。
- 遇到第一个非数字字符时就停止转换。
- `atoi("123abc")` 通常得到 `123`。
- `atoi("abc")` 通常得到 `0`。
- 它不适合需要严格错误处理的场景，更稳妥的做法通常是用 `strtol`。

## 和 `strtol` 的区别

- `atoi` 写法简单，适合快速转换。
- `strtol` 可以知道转换停止在什么位置，也更适合做严格输入校验。
- 学习阶段可以先把 `atoi` 记成“把字符串快速转成整数”的工具。

## 知识点补充

- `atoi` 本质上属于标准库中的字符串转数字函数。
- 如果你需要更可靠地判断错误，通常会配合 `[[linux系统编程/概念词条/errno|errno]]` 使用 `strtol`，而不是只用 `atoi`。
- 当输入数据来自网络、文件或用户输入时，直接用 `atoi` 要特别小心，因为它的错误信息太弱。
