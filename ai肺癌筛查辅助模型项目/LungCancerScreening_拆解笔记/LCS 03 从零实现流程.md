# LCS 03 从零实现流程

关联：[[LCS 01 项目总览]] | [[LCS 02 目录与架构]] | [[LCS 05 第一阶段 分割模块]] | [[LCS 06 第二阶段 真假结节分类]] | [[LCS 07 第三阶段 良恶性分类]] | [[LCS 08 整体推理与评估]]

## 如果从零重做这个项目

我会把整个过程分成 8 步。

### 第一步：明确任务拆分

不要直接做“癌症有无”二分类，而是拆成：

1. 找可疑区域
2. 判断是不是结节
3. 判断结节风险

### 第二步：准备数据

数据来源是 LUNA16：

1. CT 体数据
2. 结节标注
3. 候选点
4. 恶性标注

### 第三步：搭建底层数据处理

要先解决几个基础问题：

1. CT 文件怎么读
2. 坐标怎么转换
3. 怎么从完整 CT 里裁出训练样本
4. 怎么做缓存和增强

这部分主要对应 [[LCS 04 工具层与公共模块]]。

### 第四步：训练分割模型

这一步的目标不是直接给结论，而是尽量把疑似结节都找出来。

对应：[[LCS 05 第一阶段 分割模块]]

### 第五步：训练真假结节分类模型

分割会带来很多误报，所以要用 3D CNN 对候选区域二次筛选。

对应：[[LCS 06 第二阶段 真假结节分类]]

### 第六步：训练良恶性模型

在确认是真结节以后，再判断偏良性还是偏恶性。

对应：[[LCS 07 第三阶段 良恶性分类]]

### 第七步：串成完整推理链路

把三个模型串起来做：

1. 分割
2. 连通域分析
3. 候选裁块
4. 真假结节分类
5. 良恶性评估

对应：[[LCS 08 整体推理与评估]]

### 第八步：做成系统

最后再做：

1. Flask API
2. 数据库存储
3. React 页面
4. CT 切片查看器

对应：[[LCS 09 Web 后端与前端]]

## 这套流程为什么合理

因为它既兼顾了模型效果，也兼顾了工程可解释性。面试时你可以强调：这个项目不是单点算法，而是带业务流程意识的系统设计。
