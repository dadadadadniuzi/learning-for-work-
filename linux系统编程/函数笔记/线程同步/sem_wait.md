# sem_wait

> [!info] 功能
> 对信号量执行“申请资源 / P 操作 / 减一等待”。

## 函数原型

- int sem_wait([[linux系统编程/概念词条/sem_t|sem_t]] \*sem);

## 依赖头文件

- `#include <semaphore.h>`

## 输入参数

- `sem`：目标信号量对象地址。

## 输出参数

- 无直接输出参数。

## 返回值

- 成功返回 `0`。
- 失败返回 `-1`，并设置 [[linux系统编程/概念词条/errno|errno]]。

## 知识点补充

- `sem_wait` 会尝试把信号量值减一。
- 如果当前值大于 `0`，它会减一后立刻返回。
- 如果当前值等于 `0`，线程通常会阻塞，直到别的线程通过 [[linux系统编程/函数笔记/线程同步/sem_post|sem_post]] 释放资源。

## 常见用法

- 消费者等待资源。
- 控制最多只有 N 个线程同时进入某段逻辑。

## 易错点

- `sem_wait` 可能阻塞，不适合随便放在不允许卡住的路径里。
- 它和互斥锁不同，不会自动帮你保护复杂临界区逻辑。
