# 四层模型

## 它是什么

- 四层模型通常指 TCP/IP 模型，是实际网络工程和 Linux 网络编程中更常用的分层方式。
- 它把网络通信划分为应用层、传输层、网络层、网络接口层。

## 四层结构

- 应用层：应用程序和应用层协议所在位置，例如 HTTP、DNS，也包括我们自己写的 socket 应用逻辑。
- 传输层：负责端到端通信，核心协议是 [[linux网络编程/概念词条/TCP|TCP]] 和 [[linux网络编程/概念词条/UDP|UDP]]。
- 网络层：负责跨网络寻址和路由，核心协议是 IP。
- 网络接口层：负责具体链路上的帧传输，对应网卡、以太网等底层机制。

## 怎么理解

- 写 Linux 网络程序时，应用程序一般工作在应用层，通过 socket 让操作系统帮我们使用传输层和网络层能力。
- 选择 `SOCK_STREAM` 时，通常是在使用 TCP 的可靠字节流能力。
- 选择 `SOCK_DGRAM` 时，通常是在使用 UDP 的数据报能力。

## 和七层模型的关系

- 四层模型的应用层，大致对应 [[linux网络编程/概念词条/七层模型|七层模型]] 中的应用层、表示层、会话层。
- 四层模型的传输层对应七层模型的传输层。
- 四层模型的网络层对应七层模型的网络层。
- 四层模型的网络接口层，大致对应七层模型的数据链路层和物理层。

## 易错点

- 四层模型不是把七层模型“删掉几层”，而是工程实践中对职责的合并表达。
- socket 程序员通常不直接操作网络接口层，但网络接口层问题仍然会影响程序通信。
