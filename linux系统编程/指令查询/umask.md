# umask

## 它是什么

- `umask` 是“文件创建掩码”。
- 它不是文件权限本身，而是进程在创建新文件或新目录时，用来屏蔽默认权限的规则。
- 它是一个进程属性，由 shell 或程序设置。

## 它来自哪里

- 在 shell 里，`umask` 通常是一个内建命令。
- 在 C 里，对应的是 `umask(2)` 系统调用 / libc 接口。
- 子进程会继承父进程的设置。

## 它怎么看

- 查看当前值：`umask`
- 查看符号形式：`umask -S`
- 临时修改：`umask 022`

## 它怎么影响权限

- 普通文件默认基础权限通常是 `666`
- 目录默认基础权限通常是 `777`
- 最终权限会减去 `umask` 屏蔽掉的位
