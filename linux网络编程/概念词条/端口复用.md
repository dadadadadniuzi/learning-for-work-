# 端口复用

## 它是什么

- 端口复用是通过设置 socket 选项，让服务器在某些情况下可以更顺利地重新绑定同一个地址和端口。
- 网络编程课程中最常见的是通过 [[linux网络编程/函数笔记/Socket/setsockopt|setsockopt]] 设置 [[linux网络编程/概念词条/SO_REUSEADDR|SO_REUSEADDR]]。

## 怎么理解

- TCP 服务器退出后，相关连接可能还处于 `TIME_WAIT` 等状态。
- 如果马上重启服务，`bind` 可能提示地址已被使用。
- 设置端口复用可以让调试和重启更顺畅。

## 常见写法

```c
int opt = 1;
setsockopt(lfd, SOL_SOCKET, SO_REUSEADDR, &opt, sizeof(opt));
bind(lfd, (struct sockaddr *)&servaddr, sizeof(servaddr));
```

## 使用位置

- 在 `socket` 之后。
- 在 `bind` 之前。

## 易错点

- 端口复用不是万能的。如果另一个进程正在正常监听同一个地址端口，仍然可能冲突。
- `SO_REUSEADDR` 和 `SO_REUSEPORT` 不是完全相同的选项。
- 不要把“端口复用”理解成所有进程都可以随便绑定同一个端口。
