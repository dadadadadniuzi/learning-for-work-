# pid_t

## 它是什么

- `pid_t` 不是结构体。
- 它是一个用来表示进程 ID 的专用整数类型。
- 在 Linux / glibc 环境里，它通常是通过 `typedef` 定义出来的整型别名。
- 你可以把它理解成“专门装进程号的类型”，而不是普通的 `int`。

## 为什么不用普通 int

- 这样做是为了语义更清楚。
- 看到 `pid_t`，就知道这个值代表的是进程号、父进程号、进程组号或会话号一类的标识。
- 这样在 `fork`、`getpid`、`waitpid`、`kill` 等接口里更统一。

## 课程里怎么用

- `fork()` 的返回值是 `pid_t`。
- `getpid()` 返回当前进程的 `pid_t`。
- `getppid()` 返回父进程的 `pid_t`。
- `wait()` / `waitpid()` 会用 `pid_t` 指定等待哪个子进程。
- `kill()` 也用 `pid_t` 指定给哪个进程发信号。

## 常见理解

- 它本质上是一个进程标识。
- 不是进程控制块，也不是结构体。
- 进程控制块是内核里管理进程的结构，`pid_t` 只是“进程编号”这个值。

## 你需要记住的点

- `pid_t` 是类型名，不是变量名。
- 它不是结构体，没有成员可以访问。
- 在做进程相关 API 时，凡是和“进程号”相关的参数，优先联想到它。
