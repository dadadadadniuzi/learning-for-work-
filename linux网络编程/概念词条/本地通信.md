# 本地通信

## 它是什么

- 本地通信指同一台主机上的进程之间进行通信。
- 它也叫本机进程间通信，常和系统编程里的 IPC 联系起来学习。

## 常见方式

- 管道 pipe：适合有亲缘关系的进程。
- FIFO：适合无亲缘关系进程，通过路径名访问。
- mmap：通过共享内存映射交换数据。
- 信号：适合异步通知，不适合传大量数据。
- [[linux网络编程/概念词条/Unix Domain Socket|Unix Domain Socket]]：使用 socket API 做同机通信。

## 怎么理解

- 本地通信不需要跨主机，不需要路由到远程机器。
- 如果通信双方确定只在同一台机器上，本地通信方式通常更直接。
- 本地套接字兼具 IPC 和 socket 编程风格。

## 和网络通信的区别

- 本地通信范围是同一台主机。
- [[linux网络编程/概念词条/网络通信|网络通信]]可以跨主机。
- 本地通信通常不需要 IP 和端口。
