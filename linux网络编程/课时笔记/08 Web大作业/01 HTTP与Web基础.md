## 本节学什么详解

- [[linux网络编程/概念词条/HTTP|HTTP]]：应用层协议，规定浏览器和服务器之间“请求”和“响应”的文本格式。
- [[linux网络编程/概念词条/URL|URL]]：统一资源定位符，描述浏览器要访问哪个资源。
- [[linux网络编程/概念词条/Web服务器|Web 服务器]]：接收浏览器请求，返回 HTML、图片、CSS、JS 等资源。
- 浏览器：发起请求、接收响应、解析 HTML、继续请求其他资源。
- HTTP 与 TCP 的关系：HTTP 常建立在 [[linux网络编程/概念词条/TCP|TCP]] 连接之上，HTTP 关心“请求/响应长什么样”，TCP 关心“数据如何可靠送达”。

## 知识点补充

- TCP 是传输层协议，HTTP 是应用层协议，两者不是同一层的东西。
- 访问网页时，浏览器最先拿到的往往只是 HTML，之后还会继续请求页面引用的其他文件。
- Web 服务器本质上仍然是 socket 服务器，只不过应用层语义从“自己定义协议”变成了“遵守 HTTP 协议”。

## 本节内容速览

- URL 告诉浏览器“去哪儿、拿什么”。
- 浏览器通过 TCP 连接把 HTTP 请求发给服务器。
- 服务器返回 HTTP 响应。
- 浏览器解析响应并渲染页面。
