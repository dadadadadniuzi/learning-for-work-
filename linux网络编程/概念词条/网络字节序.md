# 网络字节序

## 它是什么

- 网络字节序是网络上传输多字节整数时统一采用的字节顺序。
- TCP/IP 规定网络字节序使用大端字节序。
- 常见 x86 主机通常使用小端，所以进行网络通信时需要字节序转换。

## 怎么理解

- 不同 CPU 保存整数的字节顺序可能不同。
- 如果发送方和接收方直接按各自本机格式解释多字节整数，就可能把端口号、IP 地址等解释错。
- 网络字节序相当于约定大家传输时都用同一种顺序。

## 常见转换函数

- [[linux网络编程/函数笔记/网络字节序/htonl|htonl]]：主机字节序转网络字节序，32 位。
- [[linux网络编程/函数笔记/网络字节序/htons|htons]]：主机字节序转网络字节序，16 位。
- [[linux网络编程/函数笔记/网络字节序/ntohl|ntohl]]：网络字节序转主机字节序，32 位。
- [[linux网络编程/函数笔记/网络字节序/ntohs|ntohs]]：网络字节序转主机字节序，16 位。

## 常见场景

- `sockaddr_in.sin_port = htons(port);`
- `sockaddr_in.sin_addr.s_addr = htonl(INADDR_ANY);`
- 打印客户端端口时常用 `ntohs(cliaddr.sin_port)`。

## 易错点

- `sin_port` 不能直接写普通整数端口，需要转换成网络字节序。
- 网络字节序主要针对多字节整数，单字节数据不存在字节序问题。
