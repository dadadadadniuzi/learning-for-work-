# TCP通信流程

## 它是什么

- TCP 通信流程指 TCP 客户端和服务器从创建 socket、建立连接，到收发数据、关闭连接的完整过程。

## 服务器端流程

- `socket`：创建监听 socket。
- `bind`：绑定服务器本地 IP 和端口。
- `listen`：进入监听状态。
- `accept`：等待客户端连接，成功后返回已连接 socket。
- `recv/read`：接收客户端数据。
- `send/write`：向客户端发送数据。
- `close`：关闭不再使用的 socket。

## 客户端流程

- `socket`：创建客户端 socket。
- `connect`：连接服务器 IP 和端口。
- `send/write`：发送数据。
- `recv/read`：接收服务器响应。
- `close`：关闭连接。

## 怎么理解

- 服务器的核心是“先等别人来连”。
- 客户端的核心是“主动连接服务器”。
- 连接建立后，双方就可以通过已连接 socket 双向通信。

## 易错点

- 服务器端 `accept` 返回的新 fd 才用于通信。
- 客户端 `connect` 传入的是服务器地址。
- 单连接阻塞式案例一次只能处理一个客户端。
