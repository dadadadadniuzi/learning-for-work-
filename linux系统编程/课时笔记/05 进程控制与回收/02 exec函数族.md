# exec函数族

## 本节学什么

- `execl`
- `execlp`
- `execv`
- `execvp`
- 参数传递方式与 PATH 搜索




## 本节学什么详解

- `execl`：这一节围绕这个知识点展开，重点记住它解决什么问题、什么时候用、以及和前后内容的关系。
- `execlp`：这一节围绕这个知识点展开，重点记住它解决什么问题、什么时候用、以及和前后内容的关系。
- `execv`：这一节围绕这个知识点展开，重点记住它解决什么问题、什么时候用、以及和前后内容的关系。
- `execvp`：这一节围绕这个知识点展开，重点记住它解决什么问题、什么时候用、以及和前后内容的关系。
- 参数传递方式与 PATH 搜索：这一节要分清两件事：参数到底是数组还是列表传入，以及带 `p` 的 `exec` 接口为什么能按 `PATH` 去搜索命令。

## 知识点补充
- `exec` 的本质是用新程序替换当前进程映像。
- `l` 和 `v` 区分参数组织方式，`p` 版本会搜索 `PATH`。
- 成功不会返回，失败才返回，所以失败路径要单独处理。

## 细节补充

- `wait` 是基础回收，`waitpid` 是更灵活的指定回收
- [[linux系统编程/概念词条/waitpid选项|WNOHANG]] 是非阻塞回收里最常见的选项
- `exec` 成功后不会返回，所以成功路径和失败路径要分开写
- `execl/execlp` 和 `execv/execvp` 的差别在于参数组织方式
- `fork + exec` 是创建新程序的标准套路
