# recv

> [!info] 功能
> 从已连接 socket 接收数据，常用于 TCP 通信。

## 函数原型

- `ssize_t recv(int sockfd, void *buf, size_t len, int flags);`

## 依赖头文件

- `#include <sys/types.h>`
- `#include <sys/socket.h>`

## 输入参数

- `sockfd`：已连接 socket 文件描述符。服务器端通常是 `accept` 返回的 `cfd`，客户端通常是 `connect` 成功后的 socket。
- `buf`：接收缓冲区首地址。`recv` 会把收到的数据写入这块内存，所以它必须是可写空间。
- `len`：接收缓冲区可写入的最大字节数，类型是 [[linux网络编程/概念词条/size_t|size_t]]。它应该不超过 `buf` 实际大小。
- `flags`：接收控制标志。基础阶段通常传 `0`；扩展标志见 [[linux网络编程/概念词条/send-recv标志|send-recv标志]]。

## 输出参数

- `buf`：成功时写入实际接收到的数据。

## 返回值

- 成功返回实际接收到的字节数，类型是 [[linux网络编程/概念词条/ssize_t|ssize_t]]。
- 返回 `0` 通常表示对端已经有序关闭连接。
- 失败返回 `-1`，并设置错误信息。

## 知识点补充

- `recv` 默认是阻塞的：如果没有数据到达，函数会等待。
- TCP 是字节流，`recv` 一次读到多少字节由内核缓冲区、发送情况和参数 `len` 等因素决定。
- 如果要把接收到的数据当字符串处理，需要自己考虑是否补 `\0`。
- ## 为什么不可能“读 0 字节”？
	### TCP 是流协议
	    如果：
	        连接正常
	        没有数据
	        
	    那么：
	✅ 在 **阻塞模式**：
	    会 **阻塞等待数据**
	✅ 在 **非阻塞模式**：
	    返回 `-1`
	永远不会返回0





## 常见用法

```c
char buf[1024];
ssize_t n = recv(cfd, buf, sizeof(buf), 0);
```

## 易错点

- `recv` 返回值必须检查，不能直接假设读到了完整字符串。
- 返回 `0` 是 TCP 连接关闭的重要信号。
- `buf` 不是自动以 `\0` 结尾的 C 字符串。
