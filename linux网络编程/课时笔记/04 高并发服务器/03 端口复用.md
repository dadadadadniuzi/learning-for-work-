## 本节学什么详解

- [[linux网络编程/概念词条/端口复用|端口复用]]：让服务器在某些端口仍处于系统保留或等待状态时，可以更顺利地重新绑定同一个端口，**常用于调试和服务快速重启**。
- [[linux网络编程/函数笔记/Socket/setsockopt|setsockopt]]：用于设置 socket 选项。端口复用通常在 [[linux网络编程/函数笔记/Socket/socket|socket]] 之后、[[linux网络编程/函数笔记/Socket/bind|bind]] 之前调用。
- [[linux网络编程/函数笔记/Socket/getsockopt|getsockopt]]：用于查询 socket 当前选项值。它不是开启端口复用的必要步骤，但可以用来确认 [[linux网络编程/概念词条/SO_REUSEADDR|SO_REUSEADDR]] 或 [[linux网络编程/概念词条/SO_REUSEPORT|SO_REUSEPORT]] 是否开启。
- [[linux网络编程/概念词条/SO_REUSEADDR|SO_REUSEADDR]]：常用 socket 选项，允许地址复用，缓解服务器重启后端口暂时不可绑定的问题。
- [[linux网络编程/概念词条/SO_REUSEPORT|SO_REUSEPORT]]：端口复用选项，允许多个 socket 绑定同一个 IP 和端口，常用于多 worker 负载分担。
- 端口占用排查命令：可以用 [[linux网络编程/指令查询/命令卡/ss|ss]]、[[linux网络编程/指令查询/命令卡/netstat|netstat]]、[[linux网络编程/指令查询/命令卡/lsof|lsof]] 查看哪个进程或 socket 占用了端口。

## 知识点补充

- [[linux网络编程/函数笔记/Socket/setsockopt|setsockopt]] 必须在 [[linux网络编程/函数笔记/Socket/bind|bind]] 前设置端口复用才有意义，因为 [[linux网络编程/函数笔记/Socket/bind|bind]] 才是真正占用本地 IP 和端口的动作。
- 常见代码是定义 `int opt = 1;`，再调用 `setsockopt(fd, SOL_SOCKET, SO_REUSEADDR, &opt, sizeof(opt));`。
- [[linux网络编程/概念词条/SOL_SOCKET|SOL_SOCKET]] 表示选项属于通用 socket 层；[[linux网络编程/概念词条/SO_REUSEADDR|SO_REUSEADDR]] 才是具体开启地址复用的选项。
- [[linux网络编程/概念词条/SO_REUSEADDR|SO_REUSEADDR]] 和 [[linux网络编程/概念词条/SO_REUSEPORT|SO_REUSEPORT]] 都和复用有关，但语义不同：前者更常用于服务器快速重启，后者更常用于多个 socket 共同绑定同一个端口。
- 如果 [[linux网络编程/函数笔记/Socket/bind|bind]] 失败并提示 `Address already in use`，先判断是旧连接状态影响，还是确实有进程正在监听该端口；后者要用 [[linux网络编程/指令查询/命令卡/ss|ss]]、[[linux网络编程/指令查询/命令卡/netstat|netstat]]、[[linux网络编程/指令查询/命令卡/lsof|lsof]] 排查。

## 本节内容速览

- 端口复用用于改善服务器重启绑定端口失败的问题。
- 设置入口是 [[linux网络编程/函数笔记/Socket/setsockopt|setsockopt]]。
- 查询入口是 [[linux网络编程/函数笔记/Socket/getsockopt|getsockopt]]。
- 常用选项是 [[linux网络编程/概念词条/SOL_SOCKET|SOL_SOCKET]] + [[linux网络编程/概念词条/SO_REUSEADDR|SO_REUSEADDR]]。
- 查端口用 [[linux网络编程/指令查询/命令卡/ss|ss]]、[[linux网络编程/指令查询/命令卡/netstat|netstat]]、[[linux网络编程/指令查询/命令卡/lsof|lsof]]。
