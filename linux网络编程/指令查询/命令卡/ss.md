# ss

## 命令作用

- `ss` 用于查看 socket 连接、监听端口和网络统计信息。
- 它是现代 Linux 上常用的网络排查命令，很多场景可以替代旧的 `netstat`。

## 常见写法

```bash
ss -tan
ss -tln
ss -tulpn
ss -tanp | grep 9527
```

## 参数说明

- `-t`：显示 TCP socket。
- `-u`：显示 UDP socket。
- `-l`：只显示监听状态。
- `-n`：以数字形式显示地址和端口，不做服务名解析。
- `-p`：显示关联进程信息，通常需要权限。
- `-a`：显示所有连接和监听 socket。

## 常见用途

- 查看服务器端口是否真的处于监听状态。
- 排查 `bind: Address already in use`。
- 查看哪个进程占用了某个端口。

## 示例

```bash
ss -tln | grep 9527
```

这条命令用于查看 TCP 监听端口中是否存在 `9527`。
