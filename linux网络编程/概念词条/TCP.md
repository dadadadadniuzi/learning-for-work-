# TCP

## 它是什么

- TCP 是传输层协议，全称 Transmission Control Protocol。
- 它提供面向连接、可靠、有序、基于字节流的通信能力。

## 怎么理解

- 面向连接：通信前通常需要先建立连接。
- 可靠：TCP 会通过确认、重传、序号等机制尽量保证数据可靠到达。
- 有序：接收方看到的数据顺序和发送方写入顺序一致。
- 字节流：TCP 不保留应用层每次写入的消息边界，程序需要自己处理粘包、拆包等问题。
- 报文段：TCP 在传输层会把数据组织成 [[linux网络编程/概念词条/TCP数据包格式|TCP数据包格式]] 所描述的报文段。
- 窗口：TCP 通过 [[linux网络编程/概念词条/TCP滑动窗口|TCP滑动窗口]] 支持连续发送、确认、流量控制和重传。

## 在 socket 中的典型位置

- `socket(AF_INET, SOCK_STREAM, 0)` 通常创建 TCP 套接字。
- 服务器端常见流程是 `socket -> bind -> listen -> accept -> read/write`。
- 客户端常见流程是 `socket -> connect -> read/write`。
- 建立连接时，内核会完成 [[linux网络编程/概念词条/TCP三次握手|TCP三次握手]]。
- 关闭连接时，内核会完成 [[linux网络编程/概念词条/TCP四次挥手|TCP四次挥手]]。

## 建立与关闭连接

- [[linux网络编程/概念词条/TCP三次握手|TCP三次握手]]：用于建立连接，核心是双方确认发送、接收能力，并同步初始序号。
- [[linux网络编程/概念词条/TCP四次挥手|TCP四次挥手]]：用于关闭连接，核心是 TCP 全双工通信的两个方向要分别关闭。
- [[linux网络编程/概念词条/TCP通信时序与代码对应关系|TCP通信时序与代码对应关系]]：把应用层函数调用、TCP 状态变化和网络报文方向放在同一张时序图里。
- [[linux网络编程/概念词条/TCP状态转换图|TCP状态转换图]]：从状态机角度观察 `CLOSED`、`LISTEN`、`SYN_SENT`、`ESTABLISHED`、`TIME_WAIT` 等状态如何切换。

## 可靠传输相关机制

- [[linux网络编程/概念词条/TCP数据包格式|TCP数据包格式]]：解释源端口、目的端口、序号、确认号、标志位、窗口大小等字段。
- [[linux网络编程/概念词条/TCP滑动窗口|TCP滑动窗口]]：发送方可以连续发送窗口内的数据，接收方用确认号和窗口大小反馈接收情况。
- 序号和确认号用于保证有序与可靠，窗口大小用于流量控制，重传机制用于处理丢包。

## 和 UDP 的区别

- TCP 更适合需要可靠传输、有序传输的场景。
- [[linux网络编程/概念词条/UDP|UDP]] 更轻量，但不保证可靠到达和顺序。

## 易错点

- TCP 是字节流协议，不是消息包协议。
- `write` 一次不等于对端 `read` 一次。
- TCP 的可靠性是协议层提供的，但应用层仍然要正确处理返回值、连接关闭和异常。
- `connect`、`accept` 背后涉及三次握手；`close`、`recv/read` 返回 `0` 背后涉及四次挥手。
- 应用程序通常看不到 TCP 首部；想观察 [[linux网络编程/概念词条/TCP数据包格式|TCP数据包格式]] 需要抓包。
