## 知识点补充
- 守护进程的目标是脱离终端、后台稳定运行。
- 标准模板通常是 `fork -> setsid -> chdir -> umask -> 重定向`。
- 关闭或重定向标准输入输出错误，是避免占用终端的关键。

## 本节内容速览

- 标准套路是：`fork -> 父进程退出 -> setsid -> chdir -> [[linux系统编程/指令查询/umask|umask]] -> 关闭/重定向描述符`
- 目的都是为了脱离终端、稳定运行

## 细节补充

- 守护进程的目标是脱离终端并长期稳定运行
- `setsid`、`chdir`、[[linux系统编程/指令查询/umask|umask]]、重定向是标准模板
- 组长进程不能直接创建会话，所以常先 `fork`
- 标准输入输出错误常被重定向到 `/dev/null`
- 这部分内容后面会直接影响服务器程序设计
