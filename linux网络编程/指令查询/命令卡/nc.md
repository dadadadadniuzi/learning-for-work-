# nc

## 命令作用

- `nc` 是 netcat 命令，常用于创建简单 TCP/UDP 连接、测试端口、收发文本数据。
- 在网络编程课程中，它经常用来临时充当 TCP 客户端，连接自己写的服务器程序。

## 常见写法

```bash
nc 127.0.0.1 9527
nc 192.168.1.10 8000
nc -l 9527
```

## 参数说明

- `目标IP`：要连接的服务器 IP，例如 `127.0.0.1` 表示本机。
- `目标端口`：服务器监听的端口，例如 `9527`。
- `-l`：监听模式，让 `nc` 临时充当服务端。

## 常见用途

- 测试自己写的 TCP 服务器是否能接收连接。
- 手动输入字符串，观察服务器是否能收到并回写。
- 在还没有写客户端代码时，先用 `nc` 验证服务端逻辑。

## 示例

```bash
nc 127.0.0.1 9527
```

这条命令会尝试连接本机 `9527` 端口。如果服务端正在监听，就可以在终端里输入文本进行测试。

## 易错点

- `nc` 连接失败不一定是程序错，也可能是 IP、端口、防火墙、服务端未启动等问题。
- `127.0.0.1` 只能连接本机服务，跨机器测试要使用对方真实可达 IP。
- 有些系统的 `nc` 版本不同，参数细节可能略有差异。
