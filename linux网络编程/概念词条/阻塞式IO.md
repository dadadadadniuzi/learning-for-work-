# 阻塞式IO

## 它是什么

- 阻塞式 IO 指调用某个 I/O 函数时，如果条件暂时不满足，函数会让当前执行流等待，直到条件满足或出错。

## 在 TCP 中的表现

- `accept`：没有客户端连接时会等待。
- `recv/read`：没有数据到达时会等待。
- `send/write`：发送缓冲区不可用时可能等待。

## 怎么理解

- 阻塞式 IO 写起来简单，流程接近“按顺序等待事件发生”。
- 但它容易导致程序卡在某个客户端或某个 I/O 操作上，所以后面需要多进程、多线程或 IO 多路复用来处理并发。

## 易错点

- 阻塞不是程序死掉，而是在等内核条件满足。
- 单线程阻塞式服务器如果卡在某个客户端读写上，就无法及时处理其他客户端。
- 和它相对的是 [[linux网络编程/概念词条/非阻塞I O|非阻塞I/O]]：条件不满足时立即返回，而不是一直等待。
