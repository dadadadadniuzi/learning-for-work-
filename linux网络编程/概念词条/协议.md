# 协议

## 它是什么

- 协议是一组通信规则，用来约定通信双方如何组织数据、发送数据、接收数据、解释数据和处理异常。
- 在网络编程里，协议不是某一个函数，而是操作系统协议栈和应用程序共同遵守的一套规范。

## 怎么理解

- 可以把协议理解为“通信双方共同认可的语言和流程”。
- 如果两端不遵守同一套协议，发送方发出的字节流对接收方来说就只是无意义的数据。
- 不同层次的协议负责不同事情，例如 IP 负责主机之间的寻址和转发，[[linux网络编程/概念词条/TCP|TCP]] 负责可靠传输，HTTP 负责应用层请求和响应格式。

## 常见协议层次

- 应用层协议：HTTP、FTP、DNS 等。
- 传输层协议：[[linux网络编程/概念词条/TCP|TCP]]、[[linux网络编程/概念词条/UDP|UDP]]。
- 网络层协议：IP。
- 网络接口层相关协议：以太网协议等。

## 和 socket 的关系

- `socket` 是应用程序使用网络协议栈的编程接口。
- 创建 socket 时选择的地址族、套接字类型、协议参数，会决定后续通信使用哪一类协议能力。
- 后续的 `bind`、`listen`、`accept`、`connect` 都是在协议规则下完成地址绑定、监听、连接和通信。

## 易错点

- 不要把协议理解成“函数调用顺序”。函数调用只是使用协议的一种方式。
- 不要把 TCP/IP 协议族理解成单一协议，它是一组协议的集合。
- 协议解决的是“双方如何正确通信”的问题，而不是单纯解决“能不能连上”的问题。
