# _exit

> [!info] 功能
> 立即终止当前进程，不做用户态清理。

## 函数原型

- `void _exit(int status);`

## 依赖头文件

- `#include <unistd.h>`

## 输入参数

- `status`：进程退出状态码，交给内核记录。

## 输出参数

- 无直接输出参数。

## 返回值

- 无返回值，调用后当前进程立即结束。

## 知识点补充

- `_exit` 直接进入内核退出流程，不刷新标准 IO 缓冲区，也不执行 `atexit` 回调。
- 它特别适合在 [[linux系统编程/函数笔记/进程控制/fork.md|fork]] 之后的子进程里、`exec` 失败时使用。
- 和 [[linux系统编程/函数笔记/进程控制/exit.md|exit]] 相比，`_exit` 更接近系统层面的退出。

## 常见用法

- 子进程在 `exec` 失败后立即退出。

## 易错点

- `_exit` 不会帮你做用户态收尾，调用前要明确后果。
