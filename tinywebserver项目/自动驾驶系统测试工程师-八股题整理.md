## 1. 怎么实现 C++ 和 Python 的 bridge？了解 GIL 吗？

### 题目合并

- 怎么实现的 C++ 和 Python 的 bridge
- 了解 GIL 吗

### 回答

`C++ 和 Python bridge` 的本质，是让两种语言之间可以互相调用、互相传数据。常见方式有三类：

1. `Python 调 C++`
- 最常见的是把 C++ 封装成 Python 扩展模块。
- 常用方案有 `pybind11`、`Boost.Python`、`Cython`、Python C API。
- 适合性能敏感模块，比如感知后处理、路径计算、仿真接口。

2. `C++ 调 Python`
- C++ 进程里嵌入 Python 解释器。
- 常用方式是 Python C API，例如 `Py_Initialize`、`PyImport_ImportModule`、`PyObject_CallObject`。
- 适合把 Python 当脚本层，比如配置流程、测试脚本、数据分析逻辑。

3. `进程间 bridge`
- 不直接在一个进程里混编，而是通过 `RPC`、`HTTP`、`Socket`、`gRPC`、消息队列通信。
- 好处是解耦更强，崩溃隔离更好，缺点是序列化和通信开销更大。

如果让我说“项目里怎么做 bridge”，我会这样答：

> 如果 Python 主要做测试编排、数据分析，而核心性能模块在 C++，我会优先选 `pybind11`。因为它比直接写 Python C API 更简洁，类型绑定也更自然，适合把 C++ 核心能力暴露给 Python 测试脚本调用。

### GIL 是什么

`GIL` 是 `Global Interpreter Lock`，全局解释器锁，是 CPython 解释器里的一个机制。

核心点：

- 同一个进程内，多个 Python 线程不能同时执行 Python 字节码。
- 所以 Python 多线程在 `CPU 密集型任务` 上通常不能真正并行。
- 但是在 `I/O 密集型任务` 里问题没那么大，因为线程等待 I/O 时会释放 GIL。

### 面试可直接说的版本

> C++ 和 Python 的 bridge，本质上是让两种语言互相调用。常见做法是用 `pybind11` 把 C++ 封装成 Python 模块，或者用 Python C API 在 C++ 中嵌入解释器。如果是强解耦场景，也可以走 RPC 或 Socket。  
> GIL 是 CPython 的全局解释器锁，它导致多个 Python 线程不能同时执行 Python 字节码，所以 Python 多线程更适合 I/O 密集型，不适合纯 CPU 密集型并行。

## 2. 你 C++ 重构是怎么做的？

### 回答框架

如果问“你 C++ 重构怎么做的”，不要只说“优化代码结构”，要从 `目标、步骤、风险控制、结果` 四部分答。

### 标准回答

> 我做 C++ 重构时，一般不是直接大改，而是先确认重构目标。常见目标有三类：  
> 1. 提高可维护性，比如拆掉过长函数、解耦模块依赖。  
> 2. 提高稳定性，比如统一资源管理，减少野指针、内存泄漏。  
> 3. 提高可测试性，比如把业务逻辑和 I/O、配置读取分离。  
>   
> 具体步骤上，我会先梳理现有类关系和调用链，找出高耦合点和重复逻辑；然后优先做小步重构，比如提炼函数、抽象公共类、统一接口、把裸指针替换成智能指针、引入 RAII；同时保证每一步都有编译和回归验证。  
>   
> 风险控制上，我不会一次性推翻重写，而是分阶段改，保证行为不变；对于边界逻辑会补单元测试或者回归测试。  
>   
> 如果面向性能敏感场景，我还会关注对象拷贝、锁粒度、容器选择和内存生命周期，避免为了“结构更好看”反而带来性能回退。

### 可以补充的技术点

- 用 `RAII` 管理资源
- 用 `unique_ptr/shared_ptr` 替代手动 `new/delete`
- 降低模块耦合，面向接口编程
- 拆分大类、大函数
- 补测试，保证重构不改行为

## 3. C++ 虚函数、纯虚函数

### 虚函数

`虚函数` 用来支持运行时多态。

特点：

- 基类里用 `virtual` 修饰。
- 通过基类指针或引用调用时，会在运行时决定调用子类实现。
- 底层通常依赖 `虚函数表 vtable`。

例子：

```cpp
class Base {
public:
    virtual void run() { cout << "Base"; }
};

class Child : public Base {
public:
    void run() override { cout << "Child"; }
};
```

如果：

```cpp
Base* p = new Child();
p->run();
```

最终调用的是 `Child::run()`。

### 纯虚函数

`纯虚函数` 是没有默认实现、要求子类必须重写的虚函数。

写法：

```cpp
virtual void run() = 0;
```

特点：

- 含有纯虚函数的类叫 `抽象类`
- 抽象类不能直接实例化
- 子类如果不实现纯虚函数，也还是抽象类

### 区别

- `虚函数`：可以有默认实现，子类可以重写也可以不重写
- `纯虚函数`：通常没有具体实现，子类必须实现

### 面试短答

> 虚函数是为了实现运行时多态，基类指针指向子类对象时，可以在运行时调用子类实现。纯虚函数是更强约束的虚函数，写成 `=0`，它定义接口规范，要求派生类必须实现；带纯虚函数的类就是抽象类，不能直接实例化。

## 4. B+ 树，B+ 树的分裂和合并

### B+ 树是什么

`B+ 树` 是数据库索引中最常见的数据结构，特别适合磁盘和范围查询。

特点：

- 非叶子节点只存 `键`，不存完整数据
- 叶子节点存 `键 + 数据地址`，或者直接存数据
- 所有数据都出现在叶子节点
- 叶子节点之间通常有链表，方便范围扫描

### 为什么数据库喜欢 B+ 树

- 高度低，磁盘 I/O 次数少
- 范围查询效率高
- 顺序遍历稳定

### 分裂

当某个节点插入后超过容量上限，就要 `分裂`。

过程：

1. 新建一个兄弟节点
2. 把当前节点的一半元素移动过去
3. 把中间键提升到父节点
4. 如果父节点也满了，就继续向上分裂

### 合并

删除时如果某节点元素太少，低于下限，就要 `借位或合并`。

过程：

1. 先看兄弟节点能不能借一个元素
2. 如果不能借，就和兄弟节点合并
3. 父节点对应分隔键也要删掉
4. 如果父节点也过少，就继续向上调整

### 面试短答

> B+ 树是数据库最常见的索引结构，它的非叶子节点只存键，叶子节点存完整索引项，而且叶子节点之间通常串成链表，所以非常适合范围查询。插入过满时会分裂，把中间键上推到父节点；删除过少时先尝试向兄弟借位，不够再合并，并把父节点分隔键同步调整。

## 5. MySQL 设计表之间有什么关系？MySQL 索引

### 表之间的关系

常见有四种：

1. `一对一`
- 例如用户表和用户详情表

2. `一对多`
- 例如一个订单对应多个订单项

3. `多对一`
- 本质是一对多的反向描述

4. `多对多`
- 例如学生和课程，需要中间表

设计时常见做法：

- 用 `主键` 唯一标识一行
- 用 `外键` 或逻辑关联字段表示关系
- 多对多一定要通过中间表拆开

### MySQL 索引

索引本质上是帮助数据库快速定位数据的数据结构。

常见索引：

- 主键索引
- 唯一索引
- 普通索引
- 联合索引

### 索引为什么快

- 不用全表扫描
- 通过 B+ 树快速定位到目标行或目标范围

### 联合索引注意点

最重要的是 `最左前缀原则`：

- 索引 `(a, b, c)` 能命中 `a`
- 能命中 `a,b`
- 能命中 `a,b,c`
- 不能跳过 `a` 直接高效命中 `b,c`

### 面试短答

> 表关系主要有一对一、一对多、多对多。设计时通常通过主键和外键，或者业务字段逻辑关联。MySQL 索引底层常用 B+ 树，作用是减少全表扫描，提高点查和范围查询效率。联合索引要注意最左前缀原则。

## 6. HTTP 的 GET 和 POST 区别？什么数据会放到 HTTP header 里？

### GET 和 POST 区别

1. `语义不同`
- GET 通常表示获取资源
- POST 通常表示提交数据

2. `参数位置不同`
- GET 常把参数拼在 URL 后面
- POST 通常把参数放在请求体里

3. `幂等性不同`
- GET 通常应该是幂等的
- POST 通常不保证幂等

4. `缓存和书签`
- GET 更容易被缓存、收藏、复制链接
- POST 一般不直接暴露在 URL 中

### 什么数据会放到 HTTP header

Header 里一般放 `元信息`，而不是大块业务数据。

常见包括：

- `Host`
- `Content-Type`
- `Content-Length`
- `Authorization`
- `Cookie`
- `User-Agent`
- `Accept`
- `Connection`
- `Referer`

简单理解：

- `header` 放“这次请求怎么处理”
- `body` 放“这次请求真正提交了什么内容”

### 面试短答

> GET 和 POST 最核心区别是语义和参数位置。GET 一般用于获取资源，参数常放 URL；POST 一般用于提交数据，参数常放 body。HTTP header 里通常放元信息，比如 Host、Content-Type、Authorization、Cookie、User-Agent，而真正业务数据一般放在 body 里。

## 7. Shell 语句

### 面试常见会问什么

一般不是考很深的 shell 编程，而是看你会不会基本的服务端操作。

常见会答这些：

- 查看进程：`ps -ef | grep xxx`
- 查看端口：`netstat -tunlp` 或 `ss -tunlp`
- 查看日志：`tail -f log.txt`
- 查关键字：`grep "error" log.txt`
- 统计行数：`wc -l file`
- 查看文件：`cat`、`less`
- 查文件：`find`
- 权限：`chmod`
- 压缩：`tar`

### 面试短答

> 我常用的 shell 语句主要是服务排查类，比如 `ps -ef | grep` 看进程，`tail -f` 看实时日志，`grep` 查关键字，`find` 查文件，`chmod` 改权限。如果是测试或服务端场景，shell 更多是为了快速排查问题、验证部署和分析日志。
