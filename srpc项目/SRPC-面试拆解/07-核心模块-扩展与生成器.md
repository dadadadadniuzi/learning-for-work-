## 1. `rpc_types.h`

文件：[rpc_types.h](G:\计算机学习\srpc项目\srpc-master\src\rpc_types.h)

作用：协议 traits 层。

每个协议类型声明：

1. `REQ`
2. `RESP`
3. `default_data_type`
4. `server_reply_init(...)`

意义：

模板化的 `RPCClient<RPCTYPE>` 和 `RPCServer<RPCTYPE>` 可以复用同一主流程，只在这里补协议差异。

## 2. `rpc_filter.h` 与 `rpc_module.h`

文件：

- [rpc_filter.h](G:\计算机学习\srpc项目\srpc-master\src\module\rpc_filter.h)
- [rpc_module.h](G:\计算机学习\srpc项目\srpc-master\src\module\rpc_module.h)

### `RPCFilter`

更像单个过滤逻辑。

关键函数：

1. `create_filter_task(...)`
2. `client_begin(...)`
3. `server_begin(...)`
4. `client_end(...)`
5. `server_end(...)`

### `RPCModule`

更像 filter 容器和执行器。

关键函数：

1. `add_filter(...)`
2. `client_task_begin(...)`
3. `server_task_begin(...)`
4. `client_task_end(...)`
5. `server_task_end(...)`

## 3. `rpc_trace_module.h`

文件：[rpc_trace_module.h](G:\计算机学习\srpc项目\srpc-master\src\module\rpc_trace_module.h)

作用：填充 trace 相关信息。

### `TraceModule`

提供四个生命周期接口：

1. `client_begin()`
2. `client_end()`
3. `server_begin()`
4. `server_end()`

### `RPCTraceModule`

关键函数：

1. `client_basic_info(...)`
2. `server_basic_info(...)`
3. `client_begin(...)`
4. `client_end(...)`
5. `server_begin(...)`
6. `server_end(...)`

要点：这是典型 AOP 式设计，不侵入业务逻辑。

## 4. `rpc_metrics_module.h`

文件：[rpc_metrics_module.h](G:\计算机学习\srpc项目\srpc-master\src\module\rpc_metrics_module.h)

### `RPCMetricsModule`

关键函数：

1. `client_begin(...)`

## 这一篇要记住什么

1. `rpc_types.h` 解决协议复用
2. `RPCModule + RPCFilter` 解决横切扩展
3. trace 和 metrics 都是模块化接入
4. `generator` 让使用门槛大幅降低
