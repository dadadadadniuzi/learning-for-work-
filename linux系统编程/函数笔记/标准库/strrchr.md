# strrchr

> [!info] 功能
> 在字符串中查找某个字符最后一次出现的位置。

## 函数原型

- `char *strrchr(const char *s, int c);`

## 依赖头文件

- `#include <string.h>`

## 输入参数

- `s`
  要被扫描的源字符串。
  它必须是一个合法的、以 `\0` 结尾的 C 风格字符串。

- `c`
  要查找的字符。
  虽然参数类型是 `int`，但通常传字符常量，例如 `'/'`、`'.'`、`':'`。

## 输出参数

- 无直接输出参数。

## 返回值

- 如果找到，返回指向最后一次命中字符的指针。
- 如果找不到，返回 `NULL`。

## 怎么理解

`strrchr` 和 `strchr` 的区别只有一个核心点：

- `strchr` 找第一次出现的位置
- `strrchr` 找最后一次出现的位置

例如：

```c
char *p = strrchr("/home/user/file.txt", '/');
```

它会返回最后一个 `'/'` 的位置，也就是指向 `"file.txt"` 前面的那个斜杠。

## 常见用法

```c
#include <stdio.h>
#include <string.h>

int main(void) {
    char *p = strrchr("a/b/c.txt", '/');

    if (p != NULL) {
        printf("%s\n", p + 1);
    }

    return 0;
}
```

这里会输出 `c.txt`，因为 `p + 1` 正好指向最后一个斜杠后面的文件名。

## 典型场景

- 从路径中提取最后一级文件名
- 找最后一个点号，判断扩展名
- 找最后一个分隔符

## 易错点

- 返回的是指针，不是下标。
- 找不到时返回 `NULL`，不能直接解引用。
- `strrchr` 仍然可以查找字符串结尾的 `\0`。
- 如果字符串里有多个相同字符，它返回的是最后一个，不是第一个。

## 和相关函数的区别

- `strchr`：找第一次出现的位置。
- `strrchr`：找最后一次出现的位置。
- `strpbrk`：找字符集合中的任意一个字符第一次出现的位置。

## 知识点补充

- 它非常适合路径处理和文件名处理。
- 当你想拿扩展名时，常见做法就是先找最后一个 `'.'`。
- 如果要把返回指针换算成下标，可以用 `p - s`。
