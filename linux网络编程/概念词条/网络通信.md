# 网络通信

## 它是什么

- 网络通信指进程通过网络协议栈进行通信，可以发生在同一台主机，也可以发生在不同主机之间。
- 网络编程中常见的是基于 [[linux网络编程/概念词条/TCP|TCP]] 或 [[linux网络编程/概念词条/UDP|UDP]] 的 socket 通信。

## 常见方式

- 单进程阻塞式 TCP 通信。
- [[linux网络编程/概念词条/多进程并发服务器|多进程并发服务器]]。
- [[linux网络编程/概念词条/多线程并发服务器|多线程并发服务器]]。
- [[linux网络编程/概念词条/IO多路复用|IO多路复用]]。
- [[linux网络编程/概念词条/epoll模型|epoll模型]]。
- 事件库，例如 [[linux网络编程/07 Libevent库|Libevent]]。

## 怎么理解

- IP 定位主机或网络接口。
- 端口定位主机上的服务。
- TCP/UDP 决定传输层通信语义。
- socket 是应用程序使用这些能力的接口。

## 和本地通信的区别

- 网络通信可以跨主机。
- [[linux网络编程/概念词条/本地通信|本地通信]]只发生在同一台主机内。
- 网络通信通常需要 IP、端口和传输协议。
