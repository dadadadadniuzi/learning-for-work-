# sem_init

> [!info] 功能
> 初始化一个[[linux系统编程/概念词条/sem_t|信号量对象]]。

## 函数原型

- int sem_init([[linux系统编程/概念词条/sem_t|sem_t]] \*sem, int pshared, unsigned int value);

## 依赖头文件

- `#include <semaphore.h>`

## 输入参数

- `sem`：要初始化的信号量对象地址，类型是 [[linux系统编程/概念词条/sem_t|sem_t]] `*`。
- `pshared`：是否在进程间共享。
  `0` 表示只在线程间共享，最常见。
  非 `0` 表示用于进程间共享。
- `value`：信号量初值，表示一开始可用资源数。

## 输出参数

- `sem`：初始化成功后，变成一个可用的信号量对象。

## 返回值

- 成功返回 `0`。
- 失败返回 `-1`，并设置 [[linux系统编程/概念词条/errno|errno]]。

## 知识点补充

- `sem_init` 解决的是“怎样先创建一个可用的信号量对象”这个问题。
- `value` 很关键，它决定了这个信号量更像互斥锁、资源池，还是单纯的同步通知。
- 在线程同步场景里，最常见写法是 `pshared = 0`。

## 常见用法

- 初始化线程共享信号量。
- 给生产者消费者模型准备同步对象。

## 易错点

- 不要在未初始化的信号量上直接调用 [[linux系统编程/函数笔记/线程同步/sem_wait|sem_wait]] 或 [[linux系统编程/函数笔记/线程同步/sem_post|sem_post]]。
