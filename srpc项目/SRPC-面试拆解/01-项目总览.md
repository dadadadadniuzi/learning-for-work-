# SRPC 项目总览

返回：[[SRPC-面试拆解笔记]]

## 一句话介绍

`srpc-master` 是一个基于 Workflow 的 C++ RPC 框架。  
它把开发者定义的 `protobuf/thrift` 接口自动生成成客户端和服务端代码，并在底层处理序列化、压缩、协议封装、网络传输、上下文、异步任务、trace 和 metrics。

## 这个项目解决了什么问题

1. 业务开发不必自己写 socket、收发包、包头解析。
2. 把 IDL 定义统一成简单的 client/server 使用方式。
3. 让同一份业务逻辑能跑在多种协议和传输方式上。
4. 把 trace、metrics、filter 做成可插拔横切能力。

## 用不懂网络的话来理解

可以把它理解成快递系统：

1. `IDL` 是快递单模板。
2. `代码生成器` 是自动打印单子和说明书的人。
3. `序列化` 是把对象装箱。
4. `压缩` 是压缩纸箱。
5. `协议` 是外部标签。
6. `传输` 是配送路线。
7. `server` 是收件仓库。
8. `client` 是发件员。
9. `context` 是全过程记录卡。
10. `trace/metrics/filter` 是监控和质检系统。

## 它最终实现了什么

1. 支持 `SRPC`、`bRPC`、`Thrift`、`tRPC`
2. 支持 `protobuf`、`thrift`
3. 支持 `protobuf`、`thrift binary`、`json`
4. 支持 `gzip`、`zlib`、`snappy`、`lz4`
5. 支持 `tcp`、`http` 和 ssl 变种
6. 支持异步、同步、future 风格调用
7. 支持 Workflow 任务流集成
8. 支持 trace、metrics、自定义 filter
9. 支持从 IDL 自动生成 `.srpc.h` 和 skeleton

## 最核心的本质

这个项目最重要的不是 demo，而是它搭了一套可插拔 RPC 流水线：

`IDL -> 代码生成 -> 请求对象 -> 序列化 -> 压缩 -> 协议封装 -> 传输 -> 服务端反序列化 -> 业务处理 -> 响应返回`

## 面试时建议这样开场

> 这个项目本质上不是单个业务服务，而是一套 RPC 基础设施。它把 IDL、序列化、压缩、协议、传输、上下文和横切模块分层拆开，再通过代码生成器把复杂的底层能力包装成业务开发能直接使用的接口。
